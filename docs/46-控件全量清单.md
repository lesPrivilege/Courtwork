状态：普查完成，产出活清单，供后续切单与架构拍板消化。方法：竞品拆解（WebSearch 中英文合计 22 次，覆盖 Claude Cowork/Desktop、ChatGPT Desktop/Canvas、Linear、Slack、豆包、Kimi 及桌面客户端通用惯例）× 自家结构走查（`apps/desktop/src` 实读，非只读文档）× 状态集展开。走查依据：`docs/30`（结构清单）、`docs/32-设计语言包/principles.md`（状态条/KBD/编译仪式/门禁）、`docs/45`（composer 按钮族，已定不重复）、`docs/19`（空路由判据）、`docs/41`（缺口三态）、`docs/23`（编辑面与单向编译）、`apps/desktop/SPEC.md`（现状盘点，含逐路由走查表）、`docs/09`（防呆交互）、`docs/24`（场景注册表与 skill 供给侧政策）、`docs/04`（长期 Roadmap）。

**本清单是普查产出，不是拍板记录**：标 [需架构拍板] 的项是本次普查发现的、既有 docs 未覆盖或存在张力的点，需要架构会话正式表态（同意排除 / 立项 / 修改既有拍板），普查本身无权单方面认定。

## 维护纪律

1. **活清单，非快照**：每当 apps/desktop 新增/删除一个可交互控件，或 docs/19、docs/24、docs/41 等既有拍板发生变更，本表对应行必须同步更新，不允许清单与代码/拍板长期漂移。
2. **状态列穷举，不留"等做了再想"**：新增控件进本表时必须先填完整状态集（默认/hover/加载/空/错误/禁用……），状态集不全视为未完成普查。
3. **MVP 处置只有四种诚实取值**：真实实现｜禁用态+tooltip｜明确不做（附一句理由）｜待规格（尚无拍板，不装作有）。不允许"实现了但没人知道状态集"这种隐性第五态。
4. **规格出处可追溯**：每行必须能指向一个 docs 编号或明确标"待规格"；不许凭空写处置结论。
5. **[需架构拍板] 单列不下沉**：普查会话/实现会话发现冲突只能标注上报，不能自行拍板消化，避免"实现顺手把架构决定了"。
6. **工单归属滚动更新**：新切的工单编号（P-5/P-6…、F-2…或新前缀）产出后回填对应行，避免工单表与本表脱节。

## 竞品参照速记（支撑上表处置判断的一手材料，不重复展开）

- **Claude Cowork/Desktop**：三 Tab（Chat/Cowork/Code）+ Ask before acting/Act without asking 模式选择器 + 逐文件 diff Accept/Reject + Dispatch（手机发起、桌面执行）+ Agent View（多会话仪表盘）+ Scheduled（Routines 侧栏，新建/本地/云端）+ Customize（skill/connector/plugin 统一目录+市场）+ Artifacts 面板（Copy/Download/Expand/Publish）。这套形态服务于通用后台自主 agent 心智，与 Courtwork 场景化固定编排+留人确认心智结构性不同，多数候选按此差异排除。
- **ChatGPT Canvas**：Copy/Download/Expand 三图标、无嵌套菜单；Python 代码块支持 Run；Sources 侧栏承接引用。
- **Linear**：⌘K 命令面板即产品核心；灰阶色板+状态色高对比；4px 间距网格；hover 渐进展示更多动作。
- **Slack**：消息 hover 工具条（表情/回复/更多）+ 三点菜单聚合次要动作，是"工作类工具克制平铺"范式的对照组，已在 docs/45 composer 裁定中引用同一结论。
- **豆包/Kimi**：AI 消息复制/重新生成/编辑指令三件套是中文产品收敛形态；Kimi 有独立"已归档会话"页与联网开关按钮。

## 架构裁决（2026-07-10，十项批量拍板）

1. ⌘K 命令面板：做，最小版（场景触发+案件切换+全局动作），F-2。2. 全局刷新：不做（事件流驱动无此概念），面板级动作按各自语义。3. 专注模式：做（全窗单工作面+Esc），F-2。4. 矩阵：MVP 保持手写 table，AG Grid 触发式引入（判据：S2 真实批量需列管理/虚拟滚动），落差记录非违规。5. 画布放弃修改：不做（自动保存使概念不存在），版本回退归 Stage 2。6. 新建案件：做（左栏+⌘K），F-2。7. 归档：做（popover 轻确认，可逆）；删除明确不做（数据保全+执业留痕）。8. scheduled：MVP 不做不显示（未上屏的未来能力不适用空间记忆判据），归 Stage 3。9. dispatch：MVP 不做，归 Stage 2 团队协作层。10. 下载落点：默认案件文件夹产出子目录+"在文件夹中显示"，不逐次询问，设置可改。

三发现处置：⌘K 违规由 F-2 兑现；AG Grid 落差按裁决 4 记录；callout 复制并入 F-2 第 5 项。
