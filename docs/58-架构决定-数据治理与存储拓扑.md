# 架构决定：数据治理与存储拓扑（2026-07-11）

状态：已拍板。回答四问：schema 映射存哪、如何互不污染、harness 提示词组装、turn 级用户权限与留痕。

## 一、双平面分离

- **用户可见平面** = 案件文件夹（原件/工作稿/产出）：用户的文件，用户有权动（重命名/移动/删除工作稿与产出；原件只读红线照 docs/23）。
- **应用数据平面** = app data 目录（macOS: Application Support/cn.courtwork.desktop/）：artifacts、SourceAnchor 映射、证据台账、RevisionEvent 账本、会话记录、场景状态、缓存。
- **铁律：工作语义永不放进用户文件夹**——用户误动文件不得击碎工作状态。连接靠锚：应用平面持 path + 内容 hash 指向用户文件；文件被移动→hash 重定位；找不到→礼貌降级明示"原件已移动/缺失"，绝不静默、绝不假装。

## 二、案件即目录级隔离

```
appdata/
├─ cases/<caseId>/          ← 案件即边界的物理形态
│  ├─ state.json            权威态：场景进度/事项类型标签（注册锚点，非缓存）
│  ├─ ledger.jsonl          append-only：RevisionEvent/证据/确认/turn 事件
│  ├─ artifacts/            schema 映射产物（版本化）
│  ├─ sessions/<sid>.jsonl  对话流（随案件，docs/25）
│  └─ cache/                可再生：OCR 文本层/阅读视图 md（textLayerVersion 版本化，可清除永不权威）
├─ workspaces/<id>/         同构
└─ unfiled/<sid>/           未归档对话；容器化仪式=目录迁移
```

- 数据访问层强制 caseId 作用域，**无全局查询 API**——跨案件读取结构性不存在（利冲合规的存储学表达）。demo 案件同一管线同一隔离，仅 isDemo 标记。
- **权威态 vs 缓存分家**：权威态事件溯源 append-only；缓存可随时清、清了能重建。备份/迁移单位 = 单个 case 目录（天然支持整案导出与销毁——销毁属用户不可逆决定，走确认门禁）。

## 三、harness 提示词组装（登记待完成）

系统提示词 = 优先级律（docs/53）的物化拼接：**契约段**（红线+信源纪律，core 内置常量）→ **声明段**（场景编排/uiTemplate 语境，registry 声明提供）→ **租户段**（Stage 1 词表/库路由）→ **会话段**（用户语与语料，语料标记为数据非指令）。core 按序组装，声明式、不散落代码、可快照测试（golden prompt 测试防漂移）。**工单登记：HARNESS-1（core 层，收束后第一批）**。

## 四、turn 级用户主权与留痕

| 权限 | 裁决 | 数据治理 |
|---|---|---|
| 终止 | MVP（已裁，docs/53 中断主权） | partial artifact 诚实标注，账本记 aborted 事件 |
| 重试/重连 | MVP 级批准 | 每次尝试 = 账本新事件（attempt id），旧尝试不抹除 |
| 编辑/替换历史 turn | 批准，语义 = **fork 分支** | 原 turn 与下游 artifact 留痕不可篡改；新分支新事件链；已过门禁 artifact 永不被重跑追溯删除，新版本走 RevisionEvent。排期 Stage 1（需分支 UI） |

原则一句话：**对话可以分叉，历史不可涂改**——律师工作留痕纪律在 turn 层的表达。全部数据本地、永不训练（docs/28 承诺不变）。

## 五、增补（2026-07-11）：一切容器同构

1. **容器同构定理**：cases/workspaces/unfiled 已同构；推广——任何未来容器类型（如 persona 容器：客户沟通模拟、谈判演练等角色化场景）= 同一目录形状（state/ledger/sessions/cache），请求与存储架构与卷宗完全一致。chat 的记忆、偏好、提示词按 docs/25 三层记忆落位：容器级记忆住容器目录，全局偏好住设置层——新容器类型零新机制。
2. **父目录不可删语义**：应用数据平面本不暴露给用户直接操作（双平面）；应用内容器销毁 = 用户不可逆决定走确认门禁（第二节既有）。
3. **"本地存储"表述纪律**：对外声明说真话的部分——"数据存在你的电脑上，按案件分目录隔离，不上传、不训练"（真实且可自行验证）；**不许说"绝对安全"**——本地 ≠ 加密 ≠ 不可提取，把"看起来安全"当卖点而不诚实标注边界是 docs/28 承诺体系的反面。安全增强（目录加密、钥匙串扩展）挂账 Stage 1。

## 六、增补（2026-07-11）：容器挂载一对多

1. **容器 ↔ 外部文件夹 = 一对多**（Cowork 同型）：一个容器可挂载多个用户自选的外部文件夹，挂载编排整体构成一个 worksession；挂载清单（manifest）住容器目录（即"内部 project 缓存"），锚升级为 folderId + path + hash 三元。文件夹级授权纪律照 docs/47 sources（显式选目录、永不扫全盘）。
2. **同案单容器推荐（不强制）**：同一案件推荐在一个 worksession 内处理。机制配套：新建容器挂载文件夹与既有容器重叠时轻提醒"该文件夹已在某案件中"（提示不阻止——用户可能有正当的多容器用法）。
3. **标题治理**：标题 = LLM 从请求提炼的**建议值** + 用户随时可改（案件头行内编辑既有）；提炼失败降级为文件夹名/日期，**永不因 API 问题阻塞容器创建**；改名是容器元数据轻编辑，不入 RevisionEvent 重留痕。

## 七、增补（2026-07-11）：卷宗续行块设计（HARNESS-1 核心规格）

1. **续行块 = 确定性投影，禁 LLM 压缩**：从权威态组装——state.json（阶段/类型/进度）+ 各 artifact 的声明式摘要投影（schema 声明 `rehydrationProjection`：入块字段与行预算）+ 台账尾部 N 事件 + 挂载 manifest + 未决门禁。transcript 不进块。投影可 golden test，续行质量是契约保证非模型运气。
2. **周期拍板**：卷宗级续行块随 HARNESS-1 现在做（每请求组 prompt 天然注入）；会话内 compaction 后置挂账——MVP 超窗策略 = 提示"阶段收尾开新一轮"（工作语义在 artifact，丢 transcript 损失小，结构化再水化红利）。
3. **稳定前缀纪律**：组装序按变更频率低→高 = 契约段→声明段→租户段→卷宗续行块（按 ledger seq 版本化；规范化序列化、字段序固定、易变项挤尾部，权威态不变则字节级不变）→会话增量。**优先级律语义序与缓存稳定序恰好一致**，一个排序双重目的；集中任务期 prefix cache 持续命中，对按缓存计价 provider 是直接成本杠杆。
4. 注入位置与 provider 差异（system vs 首 user 段、各家 cache 语义）归 HARNESS-1 实现细节，quirk 层处理。

5. **解耦定界（用户拍板）**：续行块机制全部通用——`rehydrationProjection` 协议、行预算规则、组装序、版本化、golden test 框架住底座（schemas/core），零法律语义；法律五 schema 只是首批投影**声明**。其他垂类 schema 各按自身体力编排投影（字段取舍与预算自定），机制零改动——与握手协议同构：换垂类=换声明。

## 八、context 经济注记（2026-07-11，用户定调）

1. **适用域论证**：本 harness 对 coding 可能不足（代码语义难投影、大量阅读爆 context）——这正是"不碰 coding"边界（docs/53）的 context 层理由；纯文字推理例外：1M context 难被文本交互撑爆、模型便宜，**优先工作语义、兼顾前缀命中**即当下最优解。现阶段任一甜点模型的 context 与推理能力完全接得住。
2. **修剪策略 roadmap（实测驱动后置）**：候选四类——turn 级 context 修剪、小模型 summarize、工具调用记录修剪、webfetch 脏 context 清理——目标是案卷推理精准度，真实数据撞墙才迭代（与 TS→Rust 同一触发纪律）。调研预案：一次派发 Grok 顺藤摸瓜社区修剪策略源码；用户既有 context 管理调研与 ecode 项目实测文档可选择性引用（coding 域结论对文字域取其形不取其值）。
3. **投递叙事位**：context 治理微调作为 roadmap 声明方向——体现底层理解能力，不体现为当期实现。

4. **席位而非许愿（用户定调总结）**：context 修剪微调、语义投影注入、卷宗 memory 管理——全部是 harness 里已设计的接口席位（本 ADR 七/八节 + docs/25），升级杠杆在我们手里，**不是完全靠 base model 兜底**。模型变强时这些席位吃红利，模型不变时它们自身就是改进路径——两头都不赌。

## 九、阶段切换语义（2026-07-11，用户模型确认入册）

1. **阶段切换 = 视图 + 声明切换，非 session 切换**：切的是 Preview renderer/场景货架/chat 取流范围（声明段）；不切的是容器/账本/路由/权威态——契约段、租户段、续行块字节级不动，**前缀命中跨阶段保持**。
2. **turn 隶属阶段**：每 turn 带 stageId；prompt 会话增量只取当前阶段对话流；跨阶段信息不回放原文 turn，走续行块结构化投影（旧阶段结论以投影几行进块）——其他阶段的对话噪音被结构性挡在 context 外，语义一行不丢。
3. 阶段归档即投影收敛：无需专门整理任务，投影随账本序号事件驱动重算（七节既有）。表层仿佛不同工作台，底层同一机器换一张声明。

## 十、总纲（2026-07-11，用户定调封顶）

**请求 API 的本质是选哪些进 prompt——本 ADR 把这个选择全程结构化了**：卷宗内 = 结构化语义投影按写好的功能逻辑规则按需注入（七/九节）；多 session、跨 agent 的重形态只出现在独立的调研/研究场景，context 完全隔离，不渗入卷宗推理。上限公理：**一个 1M context 模型还完不成的任务，加更多 context 也是添乱**——答案永远是更好的投影，不是更大的窗口。context 不是资源池，是每次都要论证的注入清单。

## 十一、chatspace 单例（2026-07-11）

chat 面的存储实体 = **单例隐形容器 chatspace/**（同构目录：sessions/cache/轻 artifacts），UI 永不暴露为对象。规则：
1. 接续 = 滚动 session：近段原文 + 旧段轻语义摘要（chat 无锚，轻摘要唯一选项，非留痕区允许）；防过拟合三线——摘要硬长度上限、只摘事实性上下文不做用户画像（docs/25 零新记忆）、用户可见可清。铁律：**chatspace 记忆永不跨入任何容器**。
2. 轻任务动词（OCR/校对/排版/粘贴）chat 面原生可用——对不建项目的用户 chat 即完整产品（docs/49 一章无容器态的常驻形态）。
3. 附件入 chatspace 缓存（随剪裁滚动、可再生级）；产出走 docs/48 既裁：工作区级默认产出目录，save=落盘+reveal，artifact 卡在流内。**文件比对话长寿**；存入仪式随时升格为容器。

## 十二、交互即事件即投影（2026-07-11）

schema 工作面的用户交互**不直接注入 prompt**，走三段链：交互 → RevisionEvent 入账本（确认/驳回/修正，带 actor/时间戳/evidenceKey/新旧值）→ artifact 状态更新 → 下次请求续行块确定性投影携带处置摘要（"已处置 3/6"）。无旁路 prompt 拼接——可 golden 测、可审计、不漂移。铁律：**用户修正是最高优先级事实，默认不触发自动重推理**（模型不许覆写用户定稿——修正即事件的对偶）；"基于修正重查"须显式动作。远期候选登记：**选区 ask**（右键选中某段/某块 → 注册表 trigger 路由微推理带锚回答，零打字）——question 卡的镜像方向，机制同源，远期不建。

## 十三、harness 微调立场与 session 本体论（2026-07-11 深夜，用户定调收口）

1. **薄 harness 定则**：一切 harness 微调必须使其更薄——内容路由归 schema，harness 只做三类事：提推理生成质量、提效率、降成本；答不出属于哪类的提案即是在塞语义，驳回。
2. **修剪负面清单式**：只修无关/污染语义，不积极压缩；连续任务保前缀命中优先于省 token。turn-level replace 与 compact/summarize 为特殊场景例外（ecode 实测形态届时取形不取值）。
3. **compact 后锚定复核**（新机制）：compaction 触发后 harness 注入显式指令——对照锚点（sourceAnchors/台账/投影）复核关键结论；把压缩风险转化为结构化自检步骤（本架构会话再水化流程的运行时同构）。
4. **session 本体论**：一容器 = 一底层 session = **高价值工作语义的优选管理体**（非 transcript 容器）；"卷宗"只是法律域临时称谓（PM=项目/招投标=标段，底层同物）。新 session/subagent 仅两情形诞生：context 需完全隔离（重型调研形态）或用户显式指定——**永不静默分裂**，用户始终知道自己在哪个语义体内。

5. **微调面收窄律（时间维度推论）**：harness 架构越迭代，微调面越窄——旋钮渐少渐锁是健康信号。维护分层随三层包体系：普通用户零维护、垂类 provider 有限维护（声明层参数）、底座 provider 独持第一性旋钮。**只有从第一性原理（效果/效率/成本三平衡）推出的微调不折旧**；针对某代模型怪癖的补丁式调优是会过期的库存，进 quirk 层且随模型退役清仓。

## 十四、外部印证注记（2026-07-12，Anthropic harness-design-long-running-apps）

六处独立收敛：生成/评审分离（自评必宽容，独立评审者可调怀疑=治理三角）；主观质量原则化评分（=de-slop 门禁化+六轴）；sprint 契约前置（=工单复述确认）；评审者亲驾应用（=验收 live 实测）；handoff artifact（=续行块）；**"harness 组件=模型做不到的假设，假设过期即拆"（=十三节薄定则原文级印证）**；"组合空间不缩小只移动"=workharness 背书。两处采纳：①MIG-1 配生成-评审对（评审 subagent 持六轴+门禁、判例校准怀疑度、5–15 轮）；②评审标准措辞纪律——措辞自带美学先验（museum quality 案），标准锚定 tokens 与原则、禁评价性形容词。
