# Trae Work 公开材料细拆（2026-07-16）

来源：docs.trae.cn 一手抓取（changelog/技能市场/记忆/Spec-Plan/沙箱/自动化），无单一「知识库开源」事件——最贴切公开物 = 技能市场 + 记忆机制 + Spec/Plan 工作流。调研原稿，不具约束力。

## 用户假设判定：「场景覆盖更高、架构仍传统通用 agent」——部分成立

- **覆盖更高属实**：Work/Code/Design 三模式 × 三端；技能市场直接搬运 Anthropic/Vercel/OpenAI/Figma 生态（14 精选中字节自研仅 3）；自动化定时任务已产品化。
- **架构传统部分成立**：技能调度 = SKILL.md 提示词驱动模型自由选用；产物 = 自由 HTML/文本（无版本化 schema、无锚点）；记忆 = 自由 markdown 笔记（AI 自动增删改）；Spec 三件套是文本纲要非机器可判定契约；确认颗粒度粗（文档级 + 命令字符串白名单），无事实等级/不可逆分级。
- **但非放养**：四层执行隔离扎实（macOS sandbox-exec / Windows 自研沙箱 / 云端独立磁盘 / worktree Git 隔离）+ 自动化执行台账——工程化程度高于纯通用 agent。

## 对投递叙事的精确化（关键修正）

差异主张继续成立：**契约先行 / 结构化 preview / 无锚不落格 / 事实等级**在其公开材料中完全空白。但战场措辞必须精确：**我们的护城河是契约与锚点，不是执行隔离**——对手沙箱是真实亮点，不可暗示其「连执行安全都没做」，否则被反打。

## 两处诚实对照

- 其 memory（自由 markdown、AI 自主增删）vs 我们 ADR-013（可撤销可审计缓存层、来源坐标、案件隔离守卫）——结构化程度我们占优，作对比素材。
- 其 scheduled 自动化已落地 vs 我们按纪律 ADR 前置（principal/idempotency/授权先行）——诚实承认「它先上了」，我们的说法是「我们不在授权语义定义前发布不可逆自动化」，这本身是卖点不是短板，但要主动讲。

## 旁证

`.trae/` 目录已成社区约定；agentskills.io 开放标准（约 40 客户端采纳）——技能分发生态在标准化，我们的场景包（descriptor 分发）与其技能（提示词分发）的本质差异=编译进契约 vs 注入提示词，正是 schema engineering 四消费者论的现实对照。
