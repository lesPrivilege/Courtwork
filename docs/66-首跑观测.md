## 采集通道(链路批交付)

1. **chat 面端到端**(产品内):连接 provider 后在 chat 面发送——发送→Rust 窄面代理(`provider_chat_request`,key 零出 Rust)→SSE 缓冲解析→回复 0ms 落格,思考内容折叠可回看。失败轮以分型文案诚实落格。
2. **「验证连接」冒烟**(产品内):设置页内嵌凭证面(#43)或首启引导卡;成功绿徽就地呈现(#44)。
3. **S3 场景真模型首跑**(脚本级,用户执行):

```bash
COURTWORK_S3_REAL=DeepSeek DEEPSEEK_API_KEY=sk-… pnpm --filter @courtwork/core demo:s3
# 或 Qwen: COURTWORK_S3_REAL=Qwen DASHSCOPE_API_KEY=…
# 或豆包:  COURTWORK_S3_REAL=豆包 ARK_API_KEY=…
```

未设 `COURTWORK_S3_REAL` 时保持 demo ScriptedProvider,行为零变。真跑输出:redline.docx 字节数、指令处理结果、事件类型序列、确认记录数、场景完成态——连同控制台原文回传即为一条观测。

4. **S3 真卷宗真跑**(FABLE-HARNESS 收官交付,2026-07-13,用户执行——LEGAL-REAL 收官通道):

```bash
COURTWORK_S3_REAL=DeepSeek DEEPSEEK_API_KEY=sk-… \
COURTWORK_S3_CONTRACT=/路径/真实合同.pdf \
  pnpm --filter @courtwork/core real:s3
```

与 demo:s3 的区别:真卷宗文件 → ReadingView → 六段组装(材料入显式数据边界) → 真 provider →
RiskList 草稿(模型出引语) → resolver 铸锚(系统出坐标) → 门禁暂停。**无 key 无全文**(provider
缺席在读材料之前拒跑);**assert-no-demo-in-real** 四项机器断言(provider 非 scripted/事件流零
demo-fixture/材料集恰为真卷宗/锚点全指真材料),违规非零退出。产出 `real-run-evidence.json`
= 真机证据七项(素材 hash/prompt hash/版本三元/模型事件/锚点命中/门禁暂停/防污染)。

## 锚点观测口径(docs/93 修订一·锚点经济性条款,2026-07-13 开口径)

每次带 citationBinding 的产出在 `artifact_produced.citationStats` 记录:
`claims`(引语总数)/`firstPassResolved`(首过公证数)/`retryRounds`(0 或 1)/
`resolvedAfterRetry`(终局公证数)/`outOfCoverage`(剪枝入缺口表数)。
**首次命中率 = firstPassResolved/claims;重试率 = retryRounds>0 的请求占比**——
此两数为「甜点档即可完成」对上游承诺的定价数据。真跑后从 events.jsonl 或
real-run-evidence.json 直读;demo(scripted)跑的 7/7 是通道自证,不作为模型数据记入。
