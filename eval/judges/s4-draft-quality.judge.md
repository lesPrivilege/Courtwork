# S4 文书起草质量 judge prompt

版本：v1（2026-07-09）

## 定位

本 judge **只评估规则评分覆盖不到的维度**。候选输出是否符合 RevisionInstructionSet
schema、定位锚点（`locator.quote`）是否真的存在于源文档、引用的法条是否真实——这些
已经由 `src/rules/schema-valid.ts`、`src/rules/revision-set-match.ts`、
`src/rules/citation-exists.ts` 做规则评分。本 judge 只回答一个问题：**在结构与依据都
成立的前提下，起草的文字本身有没有达到执业律师的写作水准**。

## 评分标准

请依据以下四项对候选输出中每条 `instructions[].text` / `annotation.text` 的文字质量
打分：

1. **文风是否贴合场景参照样本**：本案的 `dossier/01-起诉状.md`、`dossier/02-答辩状.md`
   是真实的诉讼文书文风参照——庄重、条理分明、每一句陈述后紧跟依据，不使用修辞性
   或情绪化表达。候选文字若明显偏离这种文风（过度口语化、过度铺陈修辞、或反过来
   过于简略以至于读不出论证结构），应扣分。
2. **论证是否成立**：文字是否清楚地把"事实 → 法律评价 → 结论/建议"这条链条讲完整，
   而不是只陈述结论、跳过论证过程。
3. **批注是否切中要害、可执行**：如果是 `commentOnly` 类型的批注，是否明确指出了
   审阅方接下来该做什么（例如"建议在起诉状中补充说明XX，以回应被告可能提出的YY抗辩"），
   而不是泛泛地说"这里有问题"。
4. **对己方/对方立场的措辞是否得体**：起草文书代表一方当事人的立场，但不应使用
   贬损性或不必要的攻击性措辞（例如描述对方的抗辩理由时，应使用"被告主张……"
   这类中性转述，而不是"被告狡辩……"）。

## 输出

对四项标准综合给出 0.0–1.0 的分数与是否通过（建议阈值 0.7）、以及一句话说明打分依据，
重点指出具体扣分点（不要只给"整体较好"这类空泛评语）。
