# W4 人工核验清单：Word + WPS 双端渲染

核验文件：[`test/manual-verification/sample-redline.docx`](test/manual-verification/sample-redline.docx)

该文件由 `applyRevisionInstructionSet` 对 [`test/fixtures/original.docx`](test/fixtures/original.docx)（虚构样例买卖合同）套用十条指令（[`src/test-fixtures/instruction-set.ts`](src/test-fixtures/instruction-set.ts)）生成，覆盖：条款内文字替换（含纯插入、删+插两种最小化 diff 形态）、插入新条款、整条删除、表格单元格替换、表格整行删除、纯批注不改文字，全部十条带批注，其中两条附结构化法条引用（`statuteRef`）。

我已做过的程序化核验（不需要你重复）：zip 完整性（`unzip -t`）、OOXML 结构（`w:ins`/`w:del`/`w:trPr/w:del`/`commentReference` 计数与对照）、每个管线写入的 run 是否带完整 `w:rFonts`——全部通过，详见 [`spike-report.md`](spike-report.md) 与 `src/*.test.ts`。**以下是程序无法判断、需要你用眼睛看的部分。**

## 核验项

在 **Microsoft Word** 与 **WPS 文字** 里各打开一次 `sample-redline.docx`，逐项确认。WPS 至少覆盖当前 macOS 与 Windows 桌面版；任何一端未实测时，不得宣称“WPS 兼容已验证”。

### P0 往返与结构门

- [ ] 分别用 Word 与 WPS 打开、轻改、保存副本，再让 Courtwork/reading-view 重新解析；不得报 ZIP/OOXML 损坏，也不得丢失原有正文、修订或批注。
- [ ] 同一锚点同时存在修订与批注时，两者都可见、可定位；批注回复线程在打开—保存—重开后仍完整。
- [ ] WPS 轻改保存后，既有 SourceAnchor/定位策略仍能回到原段落；不能只凭页面观感判定通过。
- [ ] 接受全部修订后重新解析，最终文本和表格结构与本清单第 6 节一致。
- [ ] 对保存前后 OOXML parts、关系文件与 namespace 做 diff；新增私有 part/namespace 必须记录并证明不破坏当前解析器，不能直接忽略。
- [ ] 中文字体回退、中西文混排和数字字体在 Word/WPS 两端一致；缺字体导致回退时必须如实记录环境，不以截图正常代替字体事实。

### 1. 修订痕迹可见且可读
- [ ] 插入的文字有下划线标记（如"违约金"条款里新插入的"五"字）
- [ ] 删除的文字有删除线标记且原文仍可读（不是直接消失）
- [ ] 修订标记的作者显示为 "Courtwork"（审阅窗格或修订气泡里可查看）
- [ ] 打开"审阅"面板，修订列表能完整列出全部改动，无"无法识别的修订"报错

### 2. 批注（10 条）
- [ ] 右侧批注气泡/批注窗格显示全部 10 条批注，文字无乱码
- [ ] 每条批注锚定的文字范围看起来对——尤其检查：
  - "违约金"条款的批注是否锚定在新插入的"五"字附近，而不是飘到别的段落
  - 第八条整条删除的批注是否锚定在被删除的整段文字上
  - 表格第三期整行删除的批注是否锚定在该行（而不是表格外）
- [ ] 附结构化法条引用（`statuteRef`）的两条批注（违约金、质保期）——本轮只核验批注文字本身正常显示，法条引用字段目前不体现在批注文本里（该字段是给未来 cite-check 工具核验用的结构化数据，不是展示字段，这里没有東西可看是预期行为，不是遗漏）

### 3. 字体（本次新增的硬性验收项）
- [ ] 选中任意一处**管线新写入的文字**（插入的"五"、"标的物"、新插入的保密条款整段等），查看字体面板：中文应显示为**仿宋_GB2312**，不是宋体、不是微软雅黑、不是"最合适字体"之类的回退提示
- [ ] 同一处文字的西文/数字部分（如有）应显示为 **Times New Roman**
- [ ] 未被本次指令触碰的原文字体保持原样（本管线不应该改动它没有编辑到的内容的字体）
- [ ] 回归病例：若出现"部分仿宋正确、部分回退成主题字体，且隔行/隔字交替"的字体错乱，先解包检查 `word/document.xml` 与 `word/comments.xml` 中**每个 `w:r`** 是否都有完整 `w:rPr/w:rFonts` 四属性（`w:ascii` + `w:eastAsia` + `w:hAnsi` + `w:cs`）。重点看未修改原文 run、`w:ins`/`w:del` 内层 run、批注引用 run、批注正文 run；任一处缺失都按阻塞处理。

### 4. 表格整行删除（已知风险点，重点看）
- [ ] "第三期"整行在开启修订标记查看时，应显示为整行带删除线（不是内容清空但边框还在、或表格结构错乱）
- [ ] 点"接受所有修订"后，该行应**从表格里实际消失**，表格从 4 行（含表头）变为 3 行，不是"文字没了但空行还占着"

### 5. 新插入段落（保密条款）
- [ ] "第七条之一 保密条款"作为新段落正确出现在"第七条 争议解决"之后、"第八条"之前（原文位置），不是插到了文档末尾或其它位置
- [ ] 该段落的插入标记正确（整段视为插入，包括其独立的段落结构，不是被并入前一段）

### 6. 接受全部修订后的最终文本
点"接受所有修订"（建议在一份副本上操作，不要覆盖这份待核验原件），确认最终文本读起来是通顺、正确的合同——重点核对这几处结果：
- 违约金比例：百分之十五
- 质保期：贰年
- 交付期限：四十五日
- 争议解决管辖：标的物所在地人民法院
- 甲方全称：星辰科技股份有限公司
- 付款比例：第一期 20%、第二期 40%，第三期整行已消失（不再有第三期）
- 新增保密条款段落存在
- 第八条原文字句已删除

### 7. Word / WPS 双端一致性
- [ ] 以上各项在 Word 与 WPS 两边观感一致，没有"Word 正常但 WPS 乱码/报损坏/修订丢失"这类落差
