# 排版与密度（排印凡例）

**发凡起例（2026-07-19 立）**：本文是产品壳字体、字重、选色的编排法——律先行，值表随 **B2-0 定值批**落 `tokens.json`（沿 B0 判例：原型提值、数学校值、门禁锁值、诚实标值）；争点见文末，未拍板项不得实现。数值以 `tokens.json` 为准。

## 发凡一 · C 基线三轨与 D 挑战案

| 轨 | 承载 | 字体 | 字重 |
|---|---|---|---|
| **标题轨** | 视图标题、章节题、hero 级 | 通行宋体（思源宋体类 OFL，具体字与字重阶待 B2-0） | ≥双字重梯度 |
| **文书轨** | 阅读视图正文、引语、修订预览（长阅读面） | 仿宋陌生化轨——**定值：朱雀仿宋（SIL OFL）**。聚珍新仿经一手条款核实**全线拒收**（2026-07-19 定谳：方正嵌入条款明文覆盖网站、计算机程序与可视化电子产品，web 与桌面内嵌同禁；拒收记录 `site/craft-evidence/SITE-CRAFT-2/juzhen-rejected/DECISION.md`）；商业采购另案可重开，token 层换字栈即可置换 | 单字重（零粗体律） |
| **功能轨** | UI chrome：按钮、标签、输入、导航、设置 | C 基线仍为现行系统 sans；D 挑战案改用非默认开放字体，须经真实 12–13px 控件与许可、覆盖、fallback 联测后另行核定 | regular / emphasized 两档（现行法） |
| **数据字** | 编号、日期、金额、表格数字 | mono 或 `tabular-nums`（现行法不变） | — |

C 是现行三轨基线，不因提出 D 自动退役。D 的举证责任在挑战方：只有明证整体效力胜出且小字号可读性不回退，才形成落地提案；同分取 C。

## R2 排印裁量边界

- 盲测矩阵只保留 **C（现行三轨）** 与 **D（全量陌生化）**；旧 A/B/C 表述作废。
- 标题轨、功能轨与数据轨由架构依据字形覆盖、CJK/Latin 基线、换行、溢出、AA、fallback、加载、包体和真实长读/小控件效力核定；D 未明证胜出即维持 C。
- 文书轨朱雀仿宋与拒系统黑体做正文是产品定讞。任何盲测结果只形成带数据提案，不构成自动授权；未经产品负责人明确改判，不得撤销文书轨。

## 字体退役律

字栈或字重改值只撤出活消费并进入机器退役账，不直接删件。`@font-face`、子集、subset manifest、SOURCE/许可链、真渲门与 craft-evidence 继续存在；退役标识进入黑名单并同步出处链。叙事文档只述角色、度量与效力差异，不传播退役字栈为可用配方。物理删除字体资产须另立架构票，R2 P2 不执行。

### P2 本轮复议门

P2 C/D 盲测结论为 C `86.5`、D `87.8`；D 的 `+1.3` 落在预锁同分区，故保 C。未来重开必须同时具备模型外盲评、真实 Tauri WKWebView 对同一 fixture 的 12–13px 权威实测，并使 D 净胜同分区且可读性不回退。本轮方向性优势不构成自动授权；文书轨朱雀仿宋仍为产品定讞。

## 发凡二 · 字重即层级

与线级语法（principles §10）同构：字重携层级语义，非装饰。功能轨维持两档；标题轨字重阶由 B2-0 实测定；**仿宋轨零粗体律**——仿宋无原生粗体，伪粗体一律禁（`font-synthesis: none` 入静态门）：文书面强调走记号系（圈点/侧点）、字号与墨色，不走加粗。避免 600/700 的中文糊重（现行法，续）。

## 发凡三 · 墨色律（文字选色）

正文即墨：文字只用 `text.*` 中性阶；彩色文字仅承载语义（语义色稀缺性裁定既有：红/琥珀仅风险、绿仅落定、**朱=人工裁决之痕——朱印/朱批皆属**，口径定谳 2026-07-19：判准是「谁把它按下去的」，朱只出现在人工裁决发生处，不作环境色）。字体×字号×字重×色**四元联测 AA**（B2 票面④：仿宋实际度量为准，不得沿黑体度量宣称）；tertiary 12px 既存缺口（三面 4.23/3.98/3.84，批次账 B2②）的闭合路径由 B2-0 在字号升档 / 值面复审 / 轨位调整间实测定谳。

## 发凡四 · 度量律

字栈与字号槽 token 化（`--font-title` / `--font-body` / `--font-ui` 类，B2-0 落 `tokens.json`）；仿宋补偿——**差在墨量不在字面**（B2-0 实测定谳 2026-07-19：1200 常用字字面积比 0.971、墨量比 0.752；「偏小偏窄」原前提被推翻），处方=行高留白为主（1.75）、字号一档为辅（16px），不得按字面率大幅升字号；CJK–西文混排配衬字显式指定（仿宋拉丁字形弱，不得裸回退）；≥16px 中文标题可轻微负字距、正文不调字距（现行法，续）；桌面壳字体随包内嵌不走网络——子集化属体积纪律，`font-display` 属站面义务。

## 发凡五 · 拉丁双轨（2026-07-19 产品定调）

西文不只有功能位。史据：磁青泥金写经本是**多文种传统**（兰札体梵文/藏文于磁青纸泥金书写），贝叶经、卷轴、经折装皆有跨文字系统先例——拉丁字母入本设计语言**无本质违和，违和只来自编排失当**。故拉丁分双轨：

- **功能配衬轨**（既定谳，不变）：文书/数据区拉丁与数字＝Times 首位配衬，只做工具位。
- **表达展示轨**（新开；字形源修订 2026-07-19·产品拍板）：hero、章题级、印记类等表达性位置的拉丁**仿写本经书**——字形源取欧洲抄经书手谱系（安色尔体/半安色尔体/卡洛林小写/人文主义书手类，Junicode 类学术写本字体为候选方向，OFL 许可纪律照旧），与中文表达位溯写经（磁青泥金）同构：**表达位取写本源（书手），功能位取刊本源（铅字）**。铅字时代通行「优雅衬线」（Garamond/Cormorant 类）明确拒于展示轨——那是 AI 路径依赖的分布中心，用它做表达位是以 slop 换 slop；**站面前卫实验田先行**，成熟经 R2 门回迁壳侧。

编排四律（展示轨的准入条件，克制审计逐处查）：①位置律——只在表达性位置，文书正文与数据区永不入；②层级律——拉丁展示不得抢中文标题的层级声部，双语同现时中文为主声部；③调和律——CJK/拉丁基线与字腔关系显式调校，非默认堆叠；④色墨律——展示拉丁选色沿既有语义（泥金只供深宗重要标题，正是写经引用的原生位），novelty 展示字与具象 kitsch（仿哥特猎奇、Papyrus 类）一律拒。总律：**陌生化的本体是编排的多样性，不是字体的猎奇**——记号系走版本学元素（界行/边栏/鱼尾，已落地），字形走写本正脉，二者同构。

## 发凡六 · 记号光学的准入边界（常设条款，2026-07-20 立）

本节把两条已裁但只存在于批次证据里的结论升格为常设法。理由：批次证据是事件件，随批归档，而归档材料无约束力——只活在证据目录里的「拒」会随归档失效，下一轮必然有人重新提案同一件事。**凡「拒」的裁定，其理由须与「准」同等地进常设文档。**

### 六之一 · 墨迹洇染：产品壳拒迁

**结论**：`feTurbulence` + `feDisplacementMap` 一类墨迹扰动滤镜，**在产品壳一律不采**。Pages（激进档）保留单点实验，落点唯一、纯几何、零色值、零渐变、零阴影、零动效；站面成熟不构成回迁授权，回迁仍须逐件重过本节。

**理由（A/B 实测后定谳，四条）**：

1. **效力不足**：B 变体只把 52px 印框边缘变毛，未增加任何裁决语义，也未改善可读性。三次 400ms 包围盒逐位相等——它连「动」都不是，只是更毛。
2. **准入词是「同时守住 AA」，不得以装饰豁免替代**。现行半透明朱印在白底的合成对比为 **2.11:1**。它靠 `aria-hidden` + 不承担唯一信息取得装饰豁免，故产品本身不构成无障碍失败；但奖级准入的措辞是「同时守住 AA」，拿豁免去换准入是偷换判据。
3. **补对比即越界**：若提高 opacity 或改色以补到 3:1，就越出了已签的朱印值与批次范围——两条路互斥，见六之二。
4. **概念成本不划算**：回迁需新增 filter 定义与对应消费门（滤镜一旦可用，就得有门管它别扩散）。收益不足以支付这笔复杂度预算，按「能删的方案优于能配置的方案」拒。

**不留半实现**：拒迁即产品源码零滤镜消费，不保留「先放着不接线」的中间态。

### 六之二 · 朱印：装饰豁免与可见性互斥

朱印当前的准入建立在一组**互斥前提**上，改任一半即同时取消另一半：

| 现状 | 依赖 |
|---|---|
| 合成对比 2.11:1 可接受 | 因为它 `aria-hidden`、是重复性裁决痕、**不承担唯一信息** |
| 它不承担唯一信息 | 因为落定语义已由法理之线与处置态文字各自完整表达 |

故：**任何以「让它更可见」为由提高 opacity、加深色值或放大尺寸的提案，都同时在取消它的豁免前提**——一旦它可见到足以承载信息，它就必须自己达标 AA，且必须回答「它表达了哪一条别处没表达的事实」。这两问同时答得出才是新提案，答不出即维持现值。

反向同理：它**不得**被接线为唯一的落定指示（如撤掉法理之线或处置文字只留印），那会让一个 2.11:1 的 `aria-hidden` 图形成为信息唯一载体。

## 四档密度

### 排印语义

| 档位 | 用途 |
|---|---|
| `reading` | 文书、修订预览、长阅读 |
| `body` | 对话、说明、按钮与普通卡片 |
| `dense` | 表格、列表、时间线与矩阵 |
| `meta` | 元信息、徽章、角标与 citation |

### 版式实用法

`meta` 是全站最小正文档位；不得把 12px 当普通正文。列表行通过 padding 形成高度，不用相邻 margin 叠加。

## 对齐

- 正式文书阅读面可以两端对齐并启用严格中文换行。
- UI、卡片、表格和对话统一左对齐，避免窄栏中文 justify 拉伸。
- 金额右对齐；同列日期统一 ISO `YYYY-MM-DD`；域编号同屏位数一致。

## KBD

KBD 是不可点击的低语提示：mono、紧凑、细描边、无 hover。一个视口内不超过四组。它不能替代可发现的按钮，也不占用主操作层级。

## 机器门清单（B2-0 随值表立门）

- **字栈单源门**：`src/` 内 `font-family` 声明必须 ∈ token 字栈声明集（仿中性色单源律形态）。
- **伪粗体门**：`font-synthesis: none` 存在锁 + 字重白名单（仿宋轨出现 >regular 字重即红）。
- **数据字门**：数据区 mono/`tabular-nums` 断言（对齐律机器化）。
- **AA 四元联测**：仿宋实际度量入 e2e（B2 票面④）。
