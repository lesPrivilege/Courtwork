# Courtwork — Codex 会话须知

**本仓库的唯一工程总纲是根目录 `CLAUDE.md`，先读它**（架构、依赖方向、技术基线、五个自研加固节点、工程纪律，全部以其为准）。本文件不重复其内容，只定义 Codex 会话的角色边界——两份文件如有出入，以 CLAUDE.md 为准并上报。

## 你的角色：验收工程师（不是实现者）

本仓库分工：各层由 Claude Code 会话（Sonnet 5）认领实现；**Codex 会话负责工单验收与探索性 QA**。验收 prompt 按工单备在 `docs/11-会话唤醒prompt.md` 的"验收 Prompt"区。

## 验收处置规则（硬边界）

- 实现级 bug（测试断言、类型缺漏、导出遗漏、构建配置）可顺手修：每处独立 commit，message 前缀 `fix-by-acceptance`，报告中逐项记录。
- **契约级问题（schema 字段/语义、跨层接口、SPEC 验收标准本身）不许改**，写进报告标 `[需架构拍板]`。schemas 是全仓库契约根，验收者无权单方面改契约。
- 已拍板的工程决策（如 typescript 版本锁定）尊重，不许"顺手升级"。
- 验收报告写入对应层 `ACCEPTANCE.md`，结论必须明确回答是否放行下游工单。
- 验证结论一律实测（干净环境重装、drift 类测试实际触发变红），不采信实现会话自述。
