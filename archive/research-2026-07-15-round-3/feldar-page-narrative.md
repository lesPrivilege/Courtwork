# feldar.com 页面叙事结构参考（2026-07-15）

来源：https://feldar.com/（AI 写作工具官网）。调研原稿，不具约束力；供 Pages 演进与终局 polish 阶段参照。视觉皮肤（白底 SaaS、hero 视频、渐变）不迁，只取结构。

## 可自然生长的模式：能力 × 微演示台账

Feldar 整站主体是约 15 个连续小节，每节 = 一个真实 UI 局部片段 + 短标签 + 一行说明（Refine / Write / Voice / Checks / Scenes / Draft / Context / Canvas / Lens / Chat / Models / Workspaces…），零 feature card、零形容词堆砌，产品自己演示自己。

与 `site-evidence-line.md` 既有裁决同构：「真实局部裁片，不重复套浏览器窗口」「连续台账与分割线」「不改成三张等权 feature card」。Feldar 验证该模式可放大——Courtwork 现有三段能力说明可长成更多台账节拍，候选素材（对应 README 的 schema 管理 + 结构化 preview 差异主张）：

- schema 表预览（descriptor 驱动通用表的真实裁片）
- 引语 → 锚点坐标跳转
- 逐条确认 / 驳回（RiskList gate）
- 修订对照与 docx 产出
- 受控提问卡片（ask-user）
- 会话窗口与只读 transcript
- non-applied 指令的确认流（OUTPUT-CONFIRM-UI-1 落地后）

既有巧思（Typer / Ghosty / Satin）保留，新增节拍复用 Ghosty 显影进场即可，不需要新效果。

## 明确不取

- hero 视频（素材生成与维护成本，且我们的证据线以静态真实裁片为准）；
- 用户证言引用（无真实用户不伪造）；
- 模型选择器展示（我们当期只注册 DeepSeek，如实呈现即可）；
- 中英混排按钮（chrome 保持完整中文产品语言，既有裁决）。

## 落点

不立即立单；作为终局 UI polish 阶段与未来 SITE 演进的叙事参照。若 Pages 台账扩容先于 polish 到来，以本笔记 + site-evidence-line 例外条款为边界另立 SITE-LEDGER 工单。
