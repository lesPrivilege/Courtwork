# 产品定位

状态：现行定本（2026-07-15）

## 一句话定位

Courtwork 是面向中国律所与企业法务的本地优先工作 agent：以案件或工作区为容器，把文件、确定性工具、模型推理、结构化产出、来源锚点和人工确认组织成可审计流程。

通用 agent 卖能力，能力随模型进步快速折旧；Courtwork 卖的是秩序——场景、字段、门禁、来源、修正与留痕。模型越便宜，这套秩序越值钱。

## 用户与问题

首要用户是需要处理大量卷宗、合同、研究材料和正式文书的律师与法务团队。他们面对的核心问题不是“模型会不会写”，而是：

- 文件分散，案件上下文难以连续；
- 结论与原文脱节，复核成本高；
- 自动化容易越权或静默降级；
- 产出无法直接进入 Word/WPS 工作流；
- 团队判断不能沉淀成稳定、可复用、可审计的形状。

## 产品形态

- `work`：容器化工作台。左侧是容器，中间是连续工作流，右侧是由 artifact schema 驱动的结构化工作面。
- `chat`：轻量单一对话画布，用于不值得建容器的任务；通过“存入”升格为容器。
- 场景由用户显式触发或由系统推荐，模型不得自行决定执行不可逆流程。
- 所有结构化结论都可回到原文锚点；所有修正都成为事件；所有正式产出都经过确认或定稿仪式。

## 滩头选择

Legal 是第一条生产垂类：它必须用真实材料证明“原件 → 引语 → 系统锚点 → 结构化结论 → 人工决定 → docx”整链，而不是以 demo 穿越替代产品证据。

为什么以 legal 切入，四条结构性理由：

1. **最符合人们对垂类的想象**——法律是公认最需要严谨的场景，可信秩序在此处的价值不需要解释；
2. **推理纯文本**——不依赖多模态与实时数据，所有字段可泛化，锚点 substrate 最成熟（文档坐标）；
3. **不依赖企业私有场景**——轻量包即可自足，接口与校验天然多源分级（法条/判例/工商，官方源与 web 线索分层），重度包留给真实驻场需求；
4. **文件级最单纯**——一套卷宗/项目材料就是完整容器，摄取、锚定、产出的链路闭合在文件层。

泛化路径也由 legal 自然引出：一次企业法务活动（如尽职调查）本身就会引入财务审计、招投标审查、产品评审类材料作为推理 context——跨包声明级调用（ADR-014 混包席位）的第一个真实拉动者是 legal 自己的工作流，不是平台想象。

## 命名叙事

Courtwork 的命名承载定位：court 昭示法律垂类——最需要严谨的场景；比 cowork 多出的 urt，读作 **you are there**——用户始终在场。这是真正的协作者工具叙事，不是自主探索代理，也不是数字员工：human in the loop 是产品承诺而非限制，员工不会被取代，判断权始终在人。这与产业用户的真实期待一致。

PM 是第二垂类的 ABI 证明：先用 schema、presentation 与 catalog 验证通用底座不依赖法律语义；真实需求、确定性计算与迁移边界成立后才创建 scenario，不能用空工作流证明泛化。

PM 垂类的现行字段与交互边界见 [PM 垂类契约](pm-vertical.md)。

以下六条是 Stage 0 的 **MVP 退出标准**，不是当前能力清单。衡量标准不是场景数量，而是一个真实材料能否走完：

1. 安全摄取并生成阅读视图；
2. 模型按场景声明推理；
3. 系统为引语铸造坐标并拒收歧义；
4. 产出通过 schema 与确认门禁；
5. 用户修正被记录并投影到续行上下文；
6. 正式文书可安全编译或修订为 docx。

后续阶段只按证据条件触发，见[产品路线图](roadmap.md)；实际成熟度、阶段和阻断只认[当前基线](../status/current.md)。

## 长期资产

- 领域 schema 与渲染词表；
- 场景声明、专业门禁和确定性接口；
- 脱敏评测集与回归基线；
- 来源解析、实体对齐、文书著录和 WPS 兼容能力；
- 经真实使用确认的 RevisionEvent 与专业判断数据。

## 明确不做

- 不把法律产品做成自主行动的通用 chatbot；
- 不以模型自证代替确定性校验；
- 不让 web 线索冒充已核验事实；
- 不自动执行不可逆动作；
- 不编辑用户上传的原件；
- 不把案件内容用于训练；
- 不在没有真实数据管线时用占位内容制造“高级感”；
- 不因平台可泛化就提前做平台市场。

## 路线原则

先打穿法律真实链路，再用第二垂类证明 ABI；先做能触红的契约和验证器，再扩场景；只有真实数据撞上性能或上下文边界时，才引入 Rust、sandbox 重构、context 修剪或更重的 agent 编排。
