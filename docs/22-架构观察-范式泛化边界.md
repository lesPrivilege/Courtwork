# 架构观察：范式泛化边界（2026-07-09）

状态：观察 + 一条纪律，非当期工单。

## 观察

本仓库的分层已在实现层面证明"垂类无关"：工具契约（判别联合/sourceId/out_of_coverage）、信源分级（A 官方权威 / B 私域维护库 / C 开放网络）、场景注册表（声明机制）、demo-data 解耦模式、SourceAnchor/RevisionEvent（可溯源 + 修正即标注）均无法律语义。**换垂类 = 五件套：新 schema 实例 + 场景 YAML 声明 + 推理 prompt + 私域维护的数据库 + 确定性接口适配器**。数据接口可以是任何私域维护的权威库（医疗诊疗指南、税则、行业标准、机构知识库），范式同构。

## 商业边界（约束，防跑偏）

平台可泛化 ≠ 现在做平台。每个垂类的价值仍在"专业团队维护的数据 + 工作流嵌入"（意见稿既定判断），平台只降低第二垂类的边际成本，不使任何垂类自动成立。顺序：法律滩头打穿 → 平台作为副产品沉淀 → 才有真选项（第二垂类自营 / 平台授权给持有私域数据的机构，后者与 Roadmap Stage 4 生态叙事同线）。

## 当期唯一纪律

**通用层（core/tools 契约/registry 机制/output 著录器/eval 底座）的命名与依赖不得渗入法律语义**；法律味只允许存在于 schemas 的类型实例、场景声明、提示词模板与 demo 语料中。实现与验收会话在通用层发现领域词汇即视为架构异味上报。
