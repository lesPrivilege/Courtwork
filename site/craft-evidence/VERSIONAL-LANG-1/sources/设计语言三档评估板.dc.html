凭什么落「激进档」

  • 字体越出功能轨：标题走通行宋体双字重，展示位起用写本拉丁（安色尔/卡洛林书手谱系）——壳侧永不允许。
  • 媒体层动效获授例外：逐字显影动 color、截图显影动 mask-position，超出四属性白名单。
  • 叙事骨架＝证据链本身（原件→引语→结论→人工确认），装饰不外溢为随机坐标或 01/02/03 脚手架。

凭什么落「中间档」——作者性向下收敛的证据

· 三层表面：L0 画布 #F7F8FA → L1 竖栏 #F2F4F7 → L1 白卡 #FFFFFF，仅左右卡消费唯一 elevation.shadow；中栏对话零投影。
· 文武线：主界（标题带/面板头/工作面外框）粗 2px 在外 + 乌丝 1px 在内（::after），次界只走乌丝——线重即层级。
· 主操作用 ink：发送/主按钮 #232B38，不占红/琥珀/蓝/绿状态预算（Vercel 黑按钮范式）。
· 裁决让位：右卡 sliver 只做导航与摘要，风险判据渲染整块交给 ③；壳侧 chat-ui-boundary 门锁零垂类字面量。

凭什么落「最克制档」

  • 版式不由人设计，从 7 基础构件（Field/Anchor/Status/Evidence/Decision/Estimate/Partial）+ 12 视图族的有限组合生长。
  • 颜色/位置/动效不许模型决定；ViewModel 冻结注入，tone 只准 5 个枚举。
  • 法理之线仅承载右栏处置状态，2px 竖线 0ms 硬切；普通卡/对话/导航一律无线。

横切发现 · 跨三档的系统性缺口

这些不属于任何单一档位，需在 GPT 前实现中统一处理。
①

字体 · 全量陌生化未落地（kill-slop）

已非 slop 的三轨：标题轨思源宋/Noto Serif SC 双字重（400/600），文书轨朱雀仿宋（零粗体律 font-synthesis:none），数据轨 mono/tabular，站面另有写本拉丁 Junicode（7 码位）+ Times/Charter 配衬。尚是 slop 的一轨：功能轨（UI chrome）两面都落系统 sans——

--font-ui: -apple-system, "Segoe UI", "PingFang SC", "MiSans", …
（site --sans 同含 -apple-system / PingFang SC）

产品已裁定：争点一（2026-07-19 夜，撤甲案）——拒苹方扩到全部轨，UI chrome 不留路径依赖字体。执行卡在 SKIN-R2 P2 盲测：

C 现行三轨 86.5 vs D 全量陌生化 87.8 Δ+1.3 落预锁同分区 → 保 C

给 GPT：功能轨 D 案候选（宋体 UI 档 / 非默认开源 sans；mono 数据轨警惕 Plex/JetBrains 类新默认脸）须过真实 Tauri WKWebView 12–13px 控件实测 + 许可/覆盖/fallback 联测，并使 D 净胜同分区且可读性不回退；字栈单源门与伪粗体门不破。文书轨朱雀仿宋是产品定谳，勿动。

②

site/ 已入磁青深宗（--bg-app:#0F1622）；产品壳仍刻本印页浅宗，themes.dark 为 B5 目标值、零消费面。

壳 · 刻本

给 GPT：切换只在 [data-theme] token 层发生，组件与版式零分支；语义 *.fg 双宗各自复算至 WCAG AA，数据区静止不因主题而异。两宗是同一语义两种纸温，不是两套设计语言。

③

原语完备度 + 记号单点纪律

十二族里 7 实现 / candidate 若干 / deferred 若干。补族要先在原生 gallery 用真实 fixture 全绘，再决定是否升生产 blueprint——不为每能力新增白卡。

记号纪律是激进档不外溢的范例：ink-bleed 滤镜全站恰 1 处消费、落定章 --motion-seal 320ms 是唯一慢于常规交互的仪式动效，均由 assert-schema-parts 断言唯一。

给 GPT：回迁唯一通货是既有 token 与设计语言；效果图新色/新阴影/弹簧动效一律回迁时剔除或先修 token，效果图不替代验收证据。
