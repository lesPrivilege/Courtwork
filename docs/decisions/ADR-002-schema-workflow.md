# ADR-002：Schema、场景与人工确认

- 状态：Accepted
- 日期：2026-07-13
- 来源：`67c66cd`、`7122346`、`32b25cc`、`328c639`、`f3864c1`、`cfa2e5a`

## 决定

- 场景是用户触发的固定声明式 pipeline；模型只在声明允许的生成节点工作。
- artifact schema、字段词表、投影、renderer 和确认策略构成同一契约。
- 模型输出必须带目标地址；系统按址校验和回填，不按自然语言或数组位置猜测。
- 合法交互是封闭集合：宣告、提问、通知、请求确认；模型不得发明系统动作。
- 所有确认、驳回与修正先形成 RevisionEvent，再改变 artifact 和续行投影。
- 缺 renderer 时使用通用只读结构视图；包缺失、过旧或拒载时显式降级。

## 后果

新增场景主要增加声明和垂类包资源。任何 external skill/MCP 接入都必须补齐 schema 与 confirmation policy；不得绕过注册表直接执行。
