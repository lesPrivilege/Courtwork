# Workharness：当场景不长飞轮，就把飞轮铺出来

**——一个法律垂类 agent 的架构宣言**

## 摘要

通用 agent 卖能力，能力在贬值；垂类 agent 卖秩序——场景、防呆、承诺、留痕——秩序在复利。本文提出 workharness：一种面向企业级文字工作的 agent 架构，其核心不是更强的模型，而是把专业判断编译为确定性结构，使效率甜点模型即可完成企业级稳定生产。

## 一、验证闭环的经济学

coding agent 一枝独秀的根本原因是编译器：模型输出获得即时、廉价、客观的对错信号，飞轮由此转动。法律、人事、采购——几乎所有企业场景——没有编译器："这份审查对不对"要靠执业判断，验证昂贵、主观、滞后。模型在这些场景没有自我改进回路。

结论不是等待更强的模型，而是把验证外置成结构：schema 校验、确定性脚本、信源锚点、确认门禁、评测集。这些朴实之物是人造验证器。场景不长飞轮，就把飞轮铺出来。

## 二、schema 三位一体

一份 schema 同时是三样东西：推理的限定（注入 prompt 时它是填空的边界——无锚不落格）、交互的界面（呈现给人时它是工作面，反馈落账本）、记忆的载体（后续请求中它是续行投影的来源）。三个时态共享一份定义，所以永远一致。

由此推论：schema 不像 prompt 那样折旧。它锚定工作语义、让人读懂结构化专业数据；上游模型变强，填空更准——协议与字段资产不随模型换代贬值，反而增值。

## 三、请求即组装

每次请求的 prompt 是五段确定性组装：契约段（红线，任何包与用户不可放宽）→ 声明段（场景编排，来自垂类包）→ 租户段 → 续行投影（工作语义的确定性摘要，禁止 LLM 压缩——投影可测试，压缩会漂移）→ 会话与语料（语料是数据不是指令）。

优先级律的语义序恰好是缓存友好的稳定序——最高优的层最稳定。这不是巧合，是切分切中了真关节的迹象。

## 四、垂类即依赖包

一个垂类 = 一个依赖包：artifact schema + 场景声明 + 提示词段 + 续行投影 + 渲染声明。机制全在底座，垂类只出声明——换垂类等于换声明，核心层长期不变。我们对六个垂类（法律、产品管理、招投标、人事、供应商、建工）做了最小实现评估：全部成立零核心改动，18 个渲染原语的封闭词表经四轮扫荡零新增。

## 五、人的位置

数字员工叙事假设人应当退出回路；我们的产品前提相反：LLM 推理节点的存在意义是协作人类。不可逆动作永远停在确认门禁；模型的更正走留痕事件不许覆写；用户修正是最高优先级事实。在人事域，个人信息保护法的自动化决策条款使"人来裁决"成为合规刚需——留人确认不是偏好，是底线。

## 六、结语

机器之强建立在不丢弃——全量数据、暴力吞吐。人有完美的丢弃函数，它被生存验证了亿万年。世界模型要求机器学会丢弃，但"丢得对不对"没有廉价验证器。在那之前，专业判断无法被学习，但可以被声明。

这就是 workharness：把路铺平，推理只放在关键语义。
