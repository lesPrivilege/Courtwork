# Courtwork 站点视觉重宣言竞稿

状态：仅供用户终审的方向稿；`SOL-FINALE` pages 段继续冻结，未修改 `Courtwork官网.dc.html`。

A/B/C 由内置 imagegen 生成；D 以当前 Courtwork 真机截图为编辑底稿、以 WorkBuddy 截图为负面参照生成。四项都只是构图与视觉语言提案，不是可发布页面，也不是真值来源。图中生成文字、案号、日期、百分比、导航、密级、条码与产品截图一律视作占位；站点落版只能使用 `docs/70-对外镜像材料/SITE-文案稿.md`、已核 demo fixture 和定版真机截图。

## 共同不变量

- 产品 UI 的藏青 / 冷灰宪法不动；暗面宣言只属于站点壳。
- 品牌 icon 保持「左侧一根竖线 + 右侧三根横杠（末杠较短）」结构，只做色域反转。
- 近黑底、高对比、mono 骨架、单信号色；零渐变、零 glow、零玻璃、零 3D 设备壳。
- 主题必须由信息结构证明「秩序对抗熵」，不能只靠末世滤镜。
- deslop 只查无项目因果的先验指纹，不因强烈、非对称或 brutalism 本身扣分。

## Prompt set

### A · 判决栅格 / Tribunal Grid

高精度 16:10 桌面站概念稿。近黑 `#080A09`、冷白、石墨线，唯一信号色酸性黄绿 `#D7FF3F`。用严密模数栅格、非对称大标题、卷宗坐标与证据截图表现制度秩序；2px 信号线是唯一彩色母题。首屏必须出现 Courtwork、`模型只生成，不裁决。`、`秩序对抗熵`、macOS 下载 CTA，并露出「成卷 / 矛盾 / 修订」连续协议带。避免暗黑 SaaS 居中 hero、圆角卡阵、假指标、glow、aurora、3D MacBook 与无意义 tech 装饰。

### B · 黑卷宗 / Black Docket

高精度 16:10 桌面站概念稿。炭黑 `#0B0B0C`、冷纸白、石墨灰，唯一信号色朱红 `#FF3B30`。把物理卷宗、页签、来源锚点与人工确认接缝翻译为网页结构；主标题中的「不裁决」由确认线揭示，右侧证据夹承接真实产品截图，下屏为连续台账而非三卡。避免 cyberpunk、终端命令行、scanline、随机 grunge、刑侦道具堆叠、装饰编号与假密级能力。

### C · 约束场 / Containment Field

高精度 16:10 桌面站概念稿。近黑蓝 `#07090D`、冷白、钢灰，唯一信号色电离蓝 `#3D66FF`。让未整理的法律碎片穿过一条有语义的 2px 边界，被编译为 `原件 / 引语 / 结论 / 确认` 四级链，再进入真实产品工作面；空间关系本身证明 harness、来源锚定与留人裁决。首屏非居中，CTA 与约束边界同体，下屏是连续的确定性链。避免粒子、glitch、matrix code、scanline、随机噪声、网络安全 dashboard 与假置信度。

### D · 藏青深栏企业形 / Deep Navy Enterprise Rail

产品壳编辑概念稿，不是站点方向。以 `apps/desktop/visual-audit/57-rp211-work-topchrome-1440.png` 为结构与语义真相源，只把完整左栏推向 `#0A2540` 锚色的 navy-900 深端；中央对话、右侧 Schema 审阅面、冷光底纸、白卡、法理线与三栏比例不动。左栏 inverse 白线按品牌 icon「一竖 + 三横、末横较短」组织，使整栏成为放大的 icon；选中 / hover / divider / disabled 都只能来自同一锚色深端派生，白色仍只表示工作纸。WorkBuddy 截图仅作负面参照：拒绝吉祥物、积分奖励、超能力话术、薄荷色、玩具圆角、促销泡泡与伙伴人格。企业血统来自秩序、信息密度和清晰收敛，不靠旧式 ERP 堆框或全局暗模。

### D 独立壳评审

**89/100，建议进入用户终审并列为修宪候选。** 与编辑底稿逐对象对拍后确认：Chat/Work、Output、Scheduled、Dispatch、Pinned、案件父行、阶段树、卷宗原件、底部用户位及中右工作面全部守恒；企业感来自同一 IA 的色面换极，不是借入新导航。评审首轮曾把底稿已有 IA 误判为新增，证据重审后已撤销该结论。

| 维度 | 分数 | 结论 |
|---|---:|---|
| 宪法 / 三栏语义守恒 | 25/25 | 对象、顺序、职责均保留 |
| 单源双极 + 白卡单义 | 18/20 | 静态观感成立，派生公式与 token 待机器证明 |
| icon 与栏结构品牌合体 | 13/15 | inverse 同形与深栏合体成立，精确同资产待源码核对 |
| 企业可信度 / 反玩具 | 15/15 | 无吉祥物、积分、伙伴人格或消费级组件语法 |
| 当代层级 / 反旧后台 | 13/15 | 扁平克制成立，仍需防通用 SAP/Jira 联想压过自身血缘 |
| 可访问与落版可证性 | 5/10 | 位图无法证明 focus、五语义色全态与 computed contrast |

最强证据：仅换左栏色面，三栏秩序与对象不动，但白色工作纸的权威性、品牌 icon 与 macOS chrome 的合体感显著增强。最大风险：深栏次级 / 三级 / 禁用前景若统一变白会压平层级，现有浅底 fg 又不能原值硬搬。一刀建议：不动对象和布局，只新增并实测 `inverse.primary / secondary / tertiary / disabled` 四级前景轨，同时逐态复核 Scheduled/Dispatch、文件副行、只读/meta、focus 和五语义色。
