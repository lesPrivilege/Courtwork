# 架构原则

状态：现行定本

## 1. 模型生成，系统裁决

模型负责提出、归纳和生成；事实等级、坐标、schema 校验、权限、预算与不可逆动作由确定性代码裁决。模型的自然语言自证不构成证据。

## 2. 无锚不落格

需要事实依据的字段必须带可验证引用。模型给出引语，系统在阅读视图中铸造坐标；歧义、缺失或越界引语经一次受限修复仍失败时，降为 `out_of_coverage`，不得编造锚点。

## 3. 留人确认

确认、定稿、文件移动、记忆写入、数据授权和任何不可逆动作都属于用户。自动化可以准备计划，不得替用户作决定。

## 4. 静默降级零容忍

未配置、覆盖不足、适配器错误、模型能力降档、包拒载和材料缺失必须显式呈现。不得用 mock、旧缓存、默认成功或空结果假装完成。

## 5. 契约先行

跨层交互先定义 schema、语义、错误和确认策略，再实现生产者与消费者。契约改变必须列出全部消费方并以全仓构建和机器搜索证明同步。

## 6. 通用机制与领域判断分离

core、registry、tools、reading-view、output 和 UI host 只保存可复用机制；法律、PM 等领域判断住垂类包。跨包绑定集中在 manifest 与装配点。

## 7. 声明优先于分支

场景、步骤、工具、投影、词表、路由与渲染形态优先表达为可校验声明。新增垂类应主要增加包数据与 renderer，不应扩散 core 特判。

## 8. 读宽写窄

读取可以吸收完整材料；写入只能通过 schema 选择性落格。厚上下文服务判断，窄输出服务可靠性，中间以投影、验证和门禁收敛。

## 9. 原件只读，历史不可涂改

用户上传材料永远只读；对话可以 fork，旧 turn、artifact 和确认记录不能被追溯抹除。新版本通过事件与版本化产物表达。

## 10. 真数据供养

没有真实数据来源的视图保持诚实空态。占位内容不得冒充完成度。测试与 demo 必须明确标识且不能进入真实路径。

## 11. 品味机器化

设计判断沉淀为 token、组件规则、封闭词表、截图和可触红门禁。审美不是一次性意见；规则必须能让后续实现低成本复用。

## 12. 触发式复杂化

Rust、sandbox、MCP、context 修剪、subagent、私域库和云协作只在数据或真实故障触发时建设。接口席位可以预留，空实现不得伪装能力。
