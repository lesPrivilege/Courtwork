# 工程工作流

状态：现行定本

## 角色

- 架构：拍板 schema 语义、跨层接口、ADR 与验收标准；不承担同工单验收。
- 实现：只实现工单和 SPEC 划定范围；跨层变化先提案。
- 验收：独立会话实测；可修实现级小缺陷，但不得改契约。

同一工单的实现者与验收者必须不同。模型名称不绑定角色，工单任命才绑定角色。

## 开工

依次读取：`CLAUDE.md`、相关 ADR、本层 `SPEC.md`、直接依赖层的公开契约。先复述范围和不变量，再写失败测试。

## TDD 与门禁

1. 用测试或静态守卫证明缺口会红；
2. 做最小实现；
3. 跑定向测试；
4. 跑包级测试与 build；
5. 完工跑全仓 `pnpm -r build`、`pnpm lint`、`pnpm test`；
6. desktop 行为变更再用隔离端口跑完整 Playwright；
7. 契约变更 grep 全部消费点。

测试数字必须来自完整实跑原始输出；`--list`、选择性运行、缓存或实现者自述不能替代验收。

## 判例（跨工单适用，逐条带出处）

### 声明与效力三判例（2026-07-19 立，源 SKIN-B2-1）

**声明在场不等于效力发生。** 三条同族，凡验证「某物已生效」时逐条自问：

1. **支持声明 ≠ 实际效力**。`CSS.supports('text-spacing-trim', …)` 回报 true，而 `trim-start` / `trim-both` / `space-all` 与 base、与阴性对照四值同宽——引擎认得这个属性名，却没实现它的行为。**装一个证不出效力的属性就是装饰**，克制审计据此拒收。
2. **分支在场 ≠ 胜出**。CSS 规则、feature flag、条件分支存在于源码里，不等于它在真实层叠/优先级/运行路径下赢得了那一票。
3. **字节到位 ≠ 渲染上身**。字体文件入仓、`@font-face` 写对、`document.fonts` 里有这一项，都不等于它真的画在了屏幕上（懒加载、回退、别名不匹配都会在中间截断）。

**共同对治**：断言效力而非断言声明，且每条效力断言旁边跟一个**必定无效的阴性对照**；对照若也显出差异，说明测量本身没有区分力，该断言不成立。

**前向红卫写法（推荐）**：暂时验证不了的项，把「当前验证不了」本身写成断言（如 `expect(CSS.supports(...)).toBe(false)`）。引擎/依赖跟上那天它自己会红，逼着改成真验证——**不留死断言，也不留无声的乐观**。同「登记不是豁免」（源 SKIN-B2-0）。

### 置换批定式：震红既有断言是世界变了（2026-07-19 立，源 SKIN-B2-1）

消费面置换批必然震红一批既有断言。**默认判定是「本批改变了世界」，不是「它们原先写错」**，故处置定式为：**按其本意重写，不放宽、不删除、不加豁免**。三个实例：

- 断言钉死的是**旧机制的形状**（「字面 serif 栈恰一处」）→ 按本意改写为新机制下的等价约束（「裸 `font-family` 零 serif 家族名 + 家族名只许住轨声明」），结果**比旧法更紧**。
- 断言钉死的是**旧定值**（`font-size: 15px`）→ 承「断关系不断值」改为断档位与比值。
- 断言依赖的**前提被新机制改变**（壳自带 `@font-face` 后 `document.fonts.check` 会把同名的所有 face 计入，其中 CSS 那份懒加载故返 false）→ 修正前提（预热真实 face），顺带把该谱从「模拟装字」升格为「验产品真装了字」。

重写后必须复验其**可红性**：注入反例仍红，才算重写而非阉割。

### 枚举必须含阅读，计数不算枚举（2026-07-19 立，源 SKIN-B2-1，补正 SKIN-B2-0「换值前先搜绑定面」）

「换值/换机制前先枚举绑定面」的前一条判例只说了要**搜**。实践证明搜到不够：本批四类绑定面全部搜到，但对其中一处只记了「1 处」未读其内容，那条断言仍由全链跑出来。**枚举的完成态是「读过每一处并判明它断言什么」，只统计出现次数不算枚举。**

### 「逐位相等」管声明处，路由目标以真渲行为锁定（2026-07-19 立，源 SKIN-B2-1 ③ 裁定）

`unicode-range` 一类「声明的覆盖面必须与文件实有覆盖面逐位相等」的判例，**适用域是凡声明 range 之处**——它防的是声明与覆盖失配，**不是命令处处声明 range**。

分野：自带 webfont 的消费点，range 是它的契约面，必须逐位相等；而**以系统字为路由目标**的消费点（如壳侧拉丁配衬走 `local()` 系统字），正确锁法是**行为级双锁**——① 子集确含目标码位（逐位相等在此成立）＋ ② 真渲证明路由确实落到目标（并以阴性对照证明测量有区分力）。

**为满足字面而新造合成字族，是拿新概念换仪式感**，复杂度节制条挡下。

### 在场 ≠ 可见（2026-07-19 立，源 SKIN-B4，补「声明与效力三判例」第四条）

`toHaveCount(1)` 只证元素**在 DOM 里**，不证它**画得出来**。SKIN-B4 的落定章四条断言全绿，
证据一摄才发现它压在标题行下、线重被缩放压到约 0.96px——**一枚看不见的印连装饰都算不上**。

同族还有：`toBeVisible()` 只查 CSS 可见性与盒子非零，不查对比度、不查遮挡、不查缩放后的线重。
**对治**：凡「视觉件已上身」类结论，断言之外必须**摄一帧亲看**；量得出的（色、线重、位）
就量出来写进证据，量不出的就承认是目视。

### 归一化让两侧可比，也让两侧各自的写法约束失守（2026-07-19 立，源 SKIN-B4）

跨面 parity 门（站 vs 壳、源 vs 编译产物、schema vs 实现）为了可比，几乎都要先归一化写法差异。
**代价是它从此看不见任何一侧的本地写法错误**：SKIN-B4 的件库单源门把 `strokeWidth=` 与
`stroke-width=` 归一后逐字比对，于是壳侧误用 HTML 式 kebab 属性——React 当场拒收、该属性
从落地起就没生效过——parity 门一路放行，只有别的谱的运行时错误守卫偶然逮到。

**对治**：每写一条归一化，就同时问「它抹掉了哪一侧的什么约束」，并把那条约束补成独立断言。
补的不是手滑，是归一化自己带进来的盲区。

### 开口子就得同时封口子（2026-07-19 立，源 SKIN-B4）

给某类件开门禁豁免时，**豁免面本身必须立刻获得等价约束**，否则「不经 A 门」会被读成「无门」。
SKIN-B4 为记号系剥出 icon 门的内联禁令（`<use>` 零几何、件库是几何唯一住所），同批即补
「件库纯度」——symbol 之外零几何元素。**豁免与封口同一枚提交落地**，中间不留窗口。

判据：写下豁免条件后自问「符合这个条件的最坏输入是什么」，把它挡住才算写完。

### 不上也要登记（2026-07-19 立，源 SKIN-B4，承 SKIN-B3「答不出即不换」）

克制审计的结论**无论是上还是不上，都必须留痕，且留在机器看得见的地方**。SKIN-B4 五记号
两上三不上，三件不上的理由逐条进门内 `UNCONSUMED` 表并**双向锁**：登记件一旦被接线即红
（逼着改登记并补审计），新件既无消费又未登记亦红。

理由与 B3 的不换清单同源：沉默的未消费件与沉默的裸 1px 同病——**不登记，下一个人无从复核
当初为何不上，只会重新纠结一遍或干脆随手接上。** 推论：**不上者无帧可摄，这不是证据缺失，
是审计结论**；验收不得以「没有截图」判为未交付。

### 排期结论不写死在纸上：互斥是运行时的锁（2026-07-20 立，源 CONVERGE-DOC-1 第二轮验收）

把「依赖顺序」与「资源互斥」压进同一个「波次」列，结果是表**自己违反了自己声明的硬约束**——两行都标着触碰同一串行文件却排在同一波，另有两张未声明的触碰者混在别处。

根因不是数错：**波次号是一个依赖未来改动面的结论，而改动面在开工前不可精确知**，把结论写死在纸上必然漂。拆成两个各自可判定的量才成立——「依赖层」（稳定、开工前可知）与「互斥标记」（逐票显式声明、不留空）；派单时取依赖已满足的票，若其互斥标记为真则确认无第二张在途。

**互斥是运行时的锁，不是纸上的分组。** 同理适用于任何「把调度结论固化进文档」的场合。

### 不为刚证明不该存在的结构立门（2026-07-20 立，源 CONVERGE-DOC-1 第二轮验收）

验收建议「同波次触碰者 ≤ 1」立机器门。但同一轮已判定波次模型本身不成立——为一个刚被废除的结构加锁，是把复杂度花在错的地方，且会让废除动作变难（门会拦着）。

判据：**加门前先问「这个结构是否应当存在」**。若答案是否，先删结构；门只加在留下来的那部分上。此例中留下来的是「每票显式声明互斥标记」——它是关于**单票**的事实，实现会话开工打开文件即知真假；而全局排期结论才是会无声腐坏的那种。

### 文档引码用符号锚，不用行号（2026-07-20 立，源 CONVERGE-DOC-1 自伤实证）

同一枚提交在文件上方插入 8 行注释，就把下方 `App.tsx:567` 推到了 `:575`——**该提交既造成漂移、又把漂移前的行号写进新文档**，而它本身正是为消除幽灵引用而设的批次。

故：文档引用代码位置一律用**符号名**（函数、常量、选择器、testid），不用行号。符号名随重构改名时会被 grep 抓到，行号则无声失效。行号只可出现在**一次性的证据记录**里（验收报告、craft-evidence），因为那些件绑定的是当时的树，不随后续提交演进。

## Git 纪律

- 多 worktree 环境下，历史复核必须用独立 worktree；共享树禁止 checkout 历史 SHA 或 stash 他人工作。
- 禁止 `git add -A`、`git add .` 和宽通配暂存。
- 逐文件显式暂存，立即检查 `git diff --cached --name-only`，然后尽快提交。
- 混合文件用 `git add -p` / cached patch + 裸 `git commit`；只有整文件完全归本会话时才可 pathspec commit。
- 不重写共享历史。误吞他人内容时前进式修正并如实记录。
- 每枚提交必须在自身 tip 上通过它触及的门禁，不能依赖未来提交补产物。

## 验收

- 使用 clean worktree 和独立 dev-server 端口；`reuseExistingServer` 不得连接共享服务。
- 验证 drift/guard 时实际注入反例并观察变红，再撤除反例恢复全绿。
- 报告写入对应层 `ACCEPTANCE.md`，明确回答是否放行。
- 清账同时满足：提交是 `main` 祖先，且 SPEC/ACCEPTANCE 有留痕。
- **验收自修探例条款（2026-07-17 立，源 PILOT-LIVE-2）**：发现实现缺陷的默认动作是驳回返修。验收会话直接修实现仅限同时满足：缺陷小且定位明确、附独立红证、全量门亲跑、提交以 `fix-by-acceptance:` 标注、架构合并前逐 diff 复核。不满足任一条即退回默认动作；此路径不得成为常态。

## 文档

工单过程不再追加到中央大册。跨层决定进 ADR；层内状态进 SPEC；验收证据进 ACCEPTANCE；全局当前态只进 `docs/status/current.md`。

**档位绑定判例（2026-07-19 立，源 SKIN-R2 P0）**：视觉 selector、组件或设计条款必须在平铺账中绑定唯一激进度档位与唯一已批提案行；同一目标双档、漏档、漏提案行都视为未获授权。档位正文只认 `docs/design/principles.md`，本文件不复制三档规则。

**归档律（2026-07-19 立，源 SKIN-B2-1 判例归档位裁定）**：**跨工单判例进本文件，单批事实进该层 `SPEC.md`。** 判据是可达性——本文件在「开工」序列里被读，下一个工单必然经过；留在某一批的 SPEC 里，下一个人不会翻到。一条经验若只对本批成立，它是事实，进 SPEC；若下一批还会踩，它是判例，进这里并带出处（`（YYYY-MM-DD 立，源 <工单>）`）。

**史料例外**：`archive/` 只存历史材料，现行文档、源码与脚本不得引用（根 `CLAUDE.md`）。**唯一例外是评审引用**——三方对照评审须逐项引原型帧位作证据链，此类引用必须标明被引件为归档史料、注明其空跑维度（若有），且结论不得依赖归档件的现行有效性。
