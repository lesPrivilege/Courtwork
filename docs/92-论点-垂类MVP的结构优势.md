# 论点：垂类 MVP 的结构优势（2026-07-10，战略终稿）

状态：经验归档（实证：意见稿到 core MVP 两天，五层验收、eval 转正、Tauri 成品）。

## 论点

1. **垂类 MVP 反而好做**：不卷通用能力（那是被上游持续打击的赛道），任务面被 harness 锁定 → 甜点区模型够用 → 无能力军备竞赛；难的部分（数据、场景 schema、纪律）不需要前沿研发，需要领域判断——团队真实经验可直接编译，上游无法商品化。
2. **核心区分点 = 场景识别硬规则 + 专属看板，且"通用不做"是激励问题而非能力问题**：有主见、维护密集、按垂类单独结账的苦活与平台玩家的横向规模激励天然错位。激励错位比能力差距更硬——能力差距被下一代模型抹平，激励错位不会。
3. **数据与场景判断线性复利 + 向上游投递**（docs/22 三资产）：上游谈接口合作更容易——我们是其 token 进入合规敏感市场的分发渠道，且回投评测与脱敏反馈。
4. **暴力碾压的蚕食诚实承认，墙立在机构层**：高认知用户用 frontier work agent DIY 会发生；但机构采购的是可培训、可标准化、可审计、有数据承诺的秩序，个体渗漏 ≠ 机构替代；渗漏者是场景挖掘的免费情报源。自足、防呆、缺口句句实话的 agent 卖的不是能力，是秩序。
5. **薄集成哲学**：task pane 薄客户端、通道网关适配器、邮件源适配器——最小实现接入各 workflow 比重集成更可靠、维护面更小（与"厚薄算法"同源：边缘做薄、判断做厚）。

## 一句话版

通用 agent 卖能力，能力在贬值；垂类 agent 卖秩序（场景、防呆、承诺、留痕），秩序在复利。
