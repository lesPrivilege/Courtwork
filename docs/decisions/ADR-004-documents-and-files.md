# ADR-004：文档生命周期与文件操作

- 状态：Accepted
- 日期：2026-07-13
- 来源：`fb1a75c`、`f432287`、`b9d5c14`、`d559678`、`f98a55c`、`1ad68d2`、`dd57a23`

## 决定

- 新文书定稿前以结构化草稿工作，用户显式“编译为 Word”；定稿后 docx 是权威表示，不回转 md。
- 客户来件 docx 从一开始走 docx 阅读/修订轨；上传原件永远只读。
- 工作稿与正式交付分轨：工作稿可轻量编辑，交付物必须有修订、确认和定稿仪式。
- 文档读取安全预检由 reading-view 提供，output 复用同一防线；不得出现安全强弱不一致的第二套解压入口。
- 文件能力分两级进入产品：无损动作可直接执行；移动/重命名先生成 FileOpsPlan，经用户选择与确认后执行并保留事务日志。删除、覆盖等销毁级动词不进入 agent 能力面。
- 原件、移动前后与举证留痕所称的“内容哈希”必须使用 SHA-256 或更强的密码学哈希；FNV/CRC 等校验和只能用于非举证内部用途，字段名、schema 注释与实现不得混用两种语义。

## 后果

Word 产物只有在文件已成功写入并可验证存在后才能标记冻结。撤销通过逆向重放实现，日志本身不可由 agent 删除。
