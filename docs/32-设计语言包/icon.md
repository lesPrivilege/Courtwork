# 应用图标（设计语言包 6/6，2026-07-09）

源稿：`icon-dark.svg`（深色底主版）+ `icon-light.svg`（浅色底版）。同一母题绘制，仅底色与前景配色互换。icns/ico 平台导出留 W9 打包，不在本次范围（docs/31）。

## 概念

母题 = **「法理之线」**：左侧一条通高竖线（卷宗侧边朱批的位置），右侧三行抽象文书行。含义直读——"有一条线立在文书旁边"，即本产品的全部主张：一切产出有边、有据、有人把关。

- **几何抽象**：全图 5 个圆角矩形，无插画、无拟物。
- **冷调**：藏青黑 `#0A2540` + 冷白 `#FAFAFA` + 钢青灰 `#7A93AC / #9DB0C4`，全部取自 tokens 色域，零暖色。
- **规避品类陈词滥调**：无天平、法槌、盾牌、立柱、书卷。法律感来自"批注线 + 文书"的工作隐喻，不来自符号库。
- 竖线刻意比文本块**上下各长出 16px**——线不是文本的一部分，是立在文本旁边的东西；这也是 UI 里法理之线通卡片全高的同构表达。
- 第三行文书行更短：文书是活的、未写完的，产品在其间工作。

## 结构参数（512 网格）

| 元素 | 位置 | 备注 |
|---|---|---|
| 底板 | 512×512, rx 116（≈22.7%） | 平台打包时按各系统 mask 重切 |
| 竖线 | x148 y124, 56×264 | 占宽 ~11%，16px 下约 1.75px，仍是"一条线" |
| 文书行 ×3 | x252, 140/140/96 宽 × 48 高，行距 44 | 16px 下 1.5px 条 + 1.4px 隙，可辨为"几行字" |

## 浅/深两版

- `icon-dark.svg`：藏青黑底 / 冷白线 / 钢青灰文本行。主版，用于浅色 dock/桌面（律所环境默认）。
- `icon-light.svg`：冷白底（加 1.5px `#EBEBEB` 描边防止白底融失）/ 藏青黑线 / 浅钢青文本行。用于深色 dock、暗色任务栏。
- 两版前景几何完全一致，只换色。禁止出现第三版配色。

## 16px 缩小可辨性自查

自查方法：SVG 直接渲染至 16×16 与 32×32 位图观察。

- **16px**：竖线约 1.75px、清晰成立；三行文本行合并观感为"右侧一叠短横"，整体读作"线+文"轮廓，与相邻图标可区分。圆角细节消失（预期内，不承载信息）。
- **32px**：三行分离可数，末行短截可辨。
- 结论：通过。若 W9 打包实测 16px 端点发虚，预授权的降级方案：16px 专版将文本行从 3 行并为 2 行（行高 64），母题不变——此调整无需回架构重议。

## 与 UI 的一致性

北极星屏左栏的产品标记（20px 内嵌 SVG）即本图标的直接缩配版（方角小 rx、同色同构），图标—界面—签名动作三者共用一个母题。后续任何 mark 变体从本源稿派生，不另起炉灶。

图标在 UI 中只是品牌母题引用，固定使用藏青/中性单色，与红/琥珀/蓝/绿/灰处置语义色系分离；任何图标都不得因状态变化而换成语义色。
