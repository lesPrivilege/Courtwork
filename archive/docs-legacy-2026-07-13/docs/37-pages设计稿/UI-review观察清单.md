# FD-1 顺带 UI review · 观察清单（登记不实施）

来源：visual-audit 截图库（重点 48/51/53/56/57/58 + 工作面 01/02/04）对照 docs/36（schema 空间分册）、docs/49（十二章）、docs/32（tokens/principles）。
性质：品味观察，登记 docs/52 批次册候选；均不构成阻塞，落版与否由架构裁量。按屏位分组，标注对照依据。

## A · 顶部 utility dock（Work 态，57/53）

- **A1 · dock 行节奏不均**：Progress / Working folders / Context 三组在横带上的间距是目测分布，组内"标签—计数—动作"的 gap 也不一致（`0/6` 与红色「继续」贴得近，`20`、`91%` 悬空）。建议将 dock 行纳入 control 凡例：组间固定 gap 档，计数一律 mono 右缘对齐。对照 docs/49 十二章"dock 坐底纸"、principles §1 间距审计。
- **A2 ·「继续」红 chip 的语义权重**：红=usage.critical 合法，但它与 Progress 计数并排时读作"进度出错"而非"上下文将满"。观察：critical 态是否应只在 Context 组内就地出现（91% 旁），不跨组挂在 Progress 侧。
- **A3 · dock 与三卡的英文/中文混排节奏**：Progress/Working folders/Context 英文（ch8 合法），但「继续」中文动词混在其间，一行内两种语言三种字重。观察项，非违规。

## B · schema 卡（修订预览，57/53）

- **B1 · citation chip 截断**：R03 详情里依据 chip 只显示「《中」即截断（57 可见）。溯源引语是信任承重件，截断到无法辨认书名时宁可换行或加宽。对照 principles §6 溯源必达、docs/52 #14 同族（矩阵截断已登记，此处是引用 chip 的同款问题）。
- **B2 · 关闭钮形制**：schema 卡右上 X 是带边框方钮，视觉重量高于同屏一切扁平控件（十二章"默认按钮全扁平，专门点出的才画框"）。X 属高频低危动作，建议降为无框 hover 显底。
- **B3 · Compare / Focus 位于 schema 面板头**：英文 chrome 动词出现在垂类工作面内部（ch8 边界：schema 空间中文语义）。属边界带——它们是宿主动作不是场景语义，但与同行的中文 Tab（时间线/矩阵审阅…）并排时语言切换突兀。登记观察，供词表分层时统一裁。

## C · chat 画布（56/58/53）

- **C1 · 空态虚线框**：58 的欢迎语容器用 dashed 描边。tokens 只有 hairline/strong 两种实线，dashed 是册外样式，且虚线占位框是社区点名的 AI tell（docs/59）。建议改文字型空态（principles 八禁 #8 同向：文字空态，不画框）。
- **C2 · Preview 收敛态右侧留白未编排**：56 中 Preview 关闭后右列只剩「Preview / Open」一行，下方大面积空底纸。三卡一纸允许右侧不展开，但此行悬在顶部像遗留物。观察：收敛态是否应并入 dock 行，右列彻底归零。
- **C3 · turn 尾 actions 行密度**：53/56 的 turn 尾（时间戳 + 复制/朗读/赞踩）图标间距与 dock 图标档不一致，且 `just now` mono 与图标基线未对齐。微观对齐项。

## D · 左栏（53/57/58）

- **D1 · 案件卡展开态的层级线**：展开后「阶段/卷宗原件·只读」两个节头与文件行仅靠缩进区分，节头与相邻行行高相同，扫读时节奏平。docs/36 左栏凡例给了"字重+小字号"通道，可再拉开一档。
- **D2 · 文件行双动作「只读 打开」**：每行常驻两个文字钮，视觉上成列噪音。docs/36 左栏凡例第 3 条"动作行内化：hover 现"——当前是常驻。登记对齐项。
- **D3 · Recent 区「存入」钮**：中文动词按钮出现在英文 chrome 区（Pinned/Recent 英文标签下），同 A3 属语言分层边界带观察。

## E · 工作面（01/02/04，RP-2.11 前旧摄，部分或已过时）

- **E1 · 矩阵列头裸编号**：04 列头为 Q1–Q5 裸字母编号，无题干缩略。docs/36 §五零编码暴露律：Q1 单独出现读不出专业含义。批次三 #14 只登记了单元格截断，列头是同屏另一处。建议列头「Q1 · 违约金」双段式。
- **E2 · 时间线来源列截断到失义**：02 的来源列全部截为「03–」，比不显示更糟（引发"这是什么"的猜测）。溯源列宁窄不残：显示件数或图标+hover，不显示残串。
- **E3 · 页脚续行按钮红字红框**：01 左下「继续本案工作」红字描边钮突出正确（usage.critical 接续行），但与状态条其余 12px 灰字并排时像错误警告。观察：critical 续行是否给 pending 琥珀而非 critical 红——文案是行动邀请，不是故障。

## F · 通表观察

- **F1 · 三处「0/6」语义重载**：左栏阶段行、dock Progress、（旧摄 51）中栏案件头都出现 `0/6`。同一数字三处重复是空间记忆资产也是噪音，建议保留左栏 + dock 两处，案件头由徽章改为文字态。
- **F2 · 头像底色**：53/57 左下头像为实心藏青圆 + 白字，是全屏唯一大面积 ink 色块（主按钮之外）。与「主操作按钮用 ink」的预算表述有轻微竞争。观察项。
