# SKIN-B2-1 · 排印置换批前后对照

皮层迁移第二批之置换半：B2-0 定的值真的上身。三轨接线、`@font-face` 别名落地、
拉丁配衬生效、排印光学实装。**UX 行为零触碰，色值零触碰。**

## 采集条件

同一采集脚本、同视口 900×620 @2x、`reducedMotion: reduce`、同一段文书语料（含中西文混排、
数字、中文标点、括号与引号，专为暴露排印问题而写）。`before/` 取自 main `2401e47`，
`after/` 取自 `impl/skin-b2-1`。

| 帧 | 内容 |
|---|---|
| `01-document-light.png` | 文书面（`.reader-pane`）：正文＋章节题＋中西文混排 |
| `02-document-dark-probe.png` | 同面的**深宗只读探针**——见下方说明 |

### 关于「双宗」帧的口径

B5 深色批**未做，本批也不做**。`02-*-dark-probe.png` 是**只读探针**：运行时把 `themes.dark`
的 token 值注入到探针容器上取一帧，**不写入任何源码、不注册 theme、不改一行产品 CSS**。
它回答的只有一个问题——**排印本身是否择纸温**。答案是不择：字栈、字号、行高、配衬与光学
全部与底色无关，深宗下同样成立。真正的深色置换仍是 B5 的活。

## 一 · 前后对照（文书面）

| | before（main） | after（本批） |
|---|---|---|
| 正文字体 | 系统 sans（苹方族） | **朱雀仿宋**（随包 GBK 子集） |
| 章节题 | 系统 sans / 字重 510 | **思源宋 SC / 字重 400**（标题轨轻端） |
| 拉丁与数字 | 系统 sans 的拉丁 | **Times New Roman**（配衬字首位接手） |
| 字号 / 行高 | 15px / 1.6 | **16px / 1.75**（墨量补偿定值） |
| 中西文间隙 | 无 | `text-autospace` 实测生效 |

肉眼可核的两处：一是正文从「工具字」变成「文书字」（仿宋的撇捺与细横一眼可辨）；
二是数字与拉丁从无衬线切到衬线，与中文同族不打架——**这正是配衬字前置的用途**。

## 二 · 机器可核（真渲探针，非自述）

| 断言 | 实测 |
|---|---|
| 文书面按名消费文书轨 | computed `font-family` ≡ `--font-body`（引号归一后逐字相等） |
| 墨量补偿上身 | `font-size` 16px，`line-height` 28px ＝ 1.75 |
| 章节题走标题轨轻端 | computed ≡ `--font-title`，`font-weight` 400 |
| chrome 留在功能轨 | `:root` computed ≡ `--font-ui`，且不含 Zhuque / Source Han Serif |
| 三件子集真上身 | `document.fonts.check` 三件皆 true；文书轨墨量 < 系统衬线 × 0.95 |
| 双字重别名分开生效 | 同族同号，600 墨量 > 400 × 1.15 |
| 数字对齐律 | 走整栈时十数字齐宽；**阴性对照**：单用 CJK 子集时确实参差 |

**CJK 不能量宽只能量墨**（承 B2-0 判例）：各 CJK 字体的字面宽一律 1em，五个汉字在朱雀、
思源、系统宋体下都是 80px——量宽分不出字体，会把「根本没换字」误判成「换成功了」。

## 三 · 排印光学：逐项过克制审计，装一件拒一件

| 项 | 实测 | 处置 |
|---|---|---|
| `text-autospace: normal` | 该串 +4px；**阴性对照（伪值）与 base 逐位相等** | **装** |
| `hanging-punctuation: allow-end` | 测试引擎 Chromium `CSS.supports` ＝ false；产品壳 WebKit 支持 | **装，但运行时效力未在本仓验证**，挂真机清单 |
| `text-spacing-trim` | `trim-start` / `trim-both` / `space-all` 与 base、与阴性对照**全部同宽 288px** | **拒**——测不出它做了任何事 |

第三行是本批的克制审计样本：`CSS.supports` 回报 true，但四个值渲染完全相同，**支持声明与实际
效力不是一回事**。装一个证不出效力的属性就是装饰，凡例的克制条挡下。若日后引擎跟上，另行带
测据补——不靠今天的乐观预支。

第二行则是**引擎限制如实登记**：e2e 断言 `CSS.supports` 当前为 false。哪天 Chromium 支持了，
这条会红，逼着把「未验证」改成真验证并撤销真机挂账——与 B2-0「登记不是豁免」同一机制。
