- 所谓“引语回跳”实际是 RiskList 行内展开 quote；production 来源按钮仍为 disabled，未打开同案
  FILE reader，也未按 SourceAnchor 高亮。
- 所谓“redline 修订 4 处”来自生产面复用的固定 demo 合同与固定计数，并非该次 RiskList 编译出的
  条文替换；现行 Legal compiler 只生成 `commentOnly`。
- “版本收尾拍板”第 2 条把“引语锚点回跳 / 修订映射 redline”写成凡例，是当时基于同一界面观察
  得出的历史裁定；这两项语义已由 ADR-010 的 2026-07-24 修订取代，当前凡例不得再引用。
- 写入的 `合同审查报告.docx` 证明文件落盘可达，但 desktop 先把 ReadingView Markdown 重建为新
  docx，再施加批注；没有把原合同 bytes 交给 output 保真链，故不证明原 DOCX 结构保留。

因此，第五、六轮在成熟度上只计“真实模型产生 RiskList + 人工处置界面可见 + docx 文件写入本案
产出目录”。来源回跳、真实 redline、原稿保真、修正终态安全与完成后账本重开须分别由
`CONTRACT-TRACE-1`、`CONTRACT-OUTPUT-TRUTH-1`、`CONTRACT-REVIEW-SAFETY-1` 实现并独立验收后
重新真机复验；六处考点也仍待正式对答案表打分。
