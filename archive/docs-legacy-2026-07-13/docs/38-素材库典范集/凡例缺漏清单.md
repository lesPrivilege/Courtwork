# 凡例缺漏清单（FD-2 交付④，2026-07-12）

> 做典范集的过程中 docs/36（+ 关联凡例 docs/32/57）盖不住的地方。逐条：**缺漏点 → 现场临时处置 → 建议补册条目**。供 SCHEMA-SPEC-1 裁决后回灌 docs/36；未裁决前，临时处置即典范集的事实基线。

## G1 · 组合页 grid 格内语义块的「块头」形制无凡例

- **缺漏**：docs/36 五级嵌套只规定 section 头（标题 510 + 分隔线）；组合页 grid 格内的语义块（如 2:1 主次栏里的表格块、核对单块）需要自己的小块头，规格无出处。
- **临时处置**：28px 块头 · bg.surface · 12px meta 文字 · 底 hairline（见 排版/06、07）——即列头凡例（listRow.headerHeight 28）的挪用。
- **建议补册**：语义块块头 = 列头同规格（28px / bg.surface / meta 12），并明确「格内语义块有块头，section 头不重复出现」的层级分工。

## G2 · statcard 内部排版无完整规格

- **缺漏**：docs/57 §2.4 只拍了 metric 28px/1.2/510 一个数；label、delta 行、卡 padding、sparkline 位全无凡例。
- **临时处置**：label meta 12 tx-3 / value metric 28 mono / delta 行 meta 12 tx-2 + 中性箭头（mono tx-3）；卡 padding 12×16，radius 6（见 排版/05、06）。
- **建议补册**：statcard 三行式解剖图（label / value / context）+ 「delta 永不消费红绿」的图示反例。

## G3 · 索证态（空态卡）的排版凡例只有一句话

- **缺漏**：docs/36 §四.3「空卷 = 索证态（场景卡引导上传）」——requiredSources 列表、缺失原因行、上传按钮位的版式无规格。
- **临时处置**：标题行（badge + 缺失原因 510）→ 说明段（sans tx-2）→ 所需材料区（verifiedBg 底纹 + mono 材料名 + 「或」分隔）→ gbtn 行（见 垂类/HR、排版/12 态一）。
- **建议补册**：索证卡四段式形制 + 「所需材料用 verifiedBg（它是 requiredSources 声明的结构化投影，非 AI 发言）」的通道归属裁决。

## G4 · schedule 临期档位的色彩阈值无拍板

- **缺漏**：docs/57 只说「临期告警消费 severity（时效属审阅面，语义色合法）」，但 ≤N 日红 / ≤M 日琥珀的阈值归属不明——凡例还是包声明？
- **临时处置**：示例按 ≤7 日红 / ≤14 日琥珀 / 其余 mono 中性（排版/07），页脚注明阈值待定。
- **建议补册**：阈值归**包声明**（法律 7/14 与 HR 合同到期 30/60 天量级不同），凡例只锁「三档封闭：红/琥珀/中性，不得多于三档」。

## G5 · matrix 单元格内多形态混排无凡例

- **缺漏**：响应性核对矩阵的单元格同列出现「响应 + 页码引用」「偏离 chip」「未提供 chip」三种形态；docs/36 的字段级凡例（mono 位/角标位）没有覆盖 matrix 单元格。
- **临时处置**：正常响应 = 12px tx-3 + mono 页码；偏离/缺漏 = severity chip；算术修正 = revision 蓝 chip（垂类/招投标）。
- **建议补册**：matrix 单元格三形态封闭集（中性文本 / 语义 chip / 修订 chip），并明确「单元格不承载法理之线，处置状态挂行级」。

## G6 · 素材库自文档行用了 11px，低于 12px 最小字号纪律

- **缺漏**：典范页脚「消费凡例」行取 mono 11px（与 KBD 11px 同档），但 typography 纪律是「12px 全站最小，以下禁用」——KBD 是唯一成文例外。
- **临时处置**：保留 11px（它是规格标注层，不是产品 UI；印刷心智里的图注）。
- **建议补册**：二选一——①册内明示「规格标注层不受产品字阶约束」；②素材库自文档行升 12px。倾向①，但需架构过目。

## G7 · narrative 行内 verified 值的形制无凡例

- **缺漏**：principles §11 的 verified 通道只有块级底纹规格（radius.sm 底纹区）；narrative 句子里内嵌的核验数字（「判赔 ¥834,000」）需要行内形制。
- **临时处置**：行内 mono + verifiedBg + padding 0 4px + radius 4（排版/10）。
- **建议补册**：行内 verified 片段规格（与块级底纹同源），并限定只用于数字/短引语，整句引语必须落块级底纹区。

## G8 · 零编码暴露映射表的「可见位」无凡例

- **缺漏**：docs/36 §五要求机器枚举映射为专业词、映射表随包声明，但用户在界面上从哪里看到「未覆盖 · 需补材料」的口径解释（tooltip？图例？）无规定。
- **临时处置**：chip 的 title 属性承载一句话口径（各垂类页）。
- **建议补册**：映射词的口径解释统一挂 title/浮层（120ms hover 凡例内），不占版面；映射表全文只在包文档，不进 UI。

## G9 · compare 原语字段列与「相同字段」的退后处置无凡例

- **缺漏**：docs/57 compare 只定义了 diff 高亮复用修订双色；字段名列宽、相同字段的视觉退后、「只看差异」的默认态皆无。
- **临时处置**：字段列 128px meta 12 tx-3；相同字段 text.secondary；筛选默认「只看差异」且筛选态常显（排版/08）。
- **建议补册**：compare 三列式（字段名 / 左 / 右）+ 相同字段退后 + 默认筛选态三条一并成文。

## G10 · 多宿主并排取证页（画廊/三态对照）是素材库私有形态

- **缺漏**：docs/36 只有单 PreviewHost 语法；画廊页、三态并排页在产品中不存在。
- **临时处置**：素材库内部形态：bg.app 地面 + 多张 L1 白卡并置 + mono 态标签（垂类/画廊、排版/12）。
- **建议补册**：**不补**——标注为素材库私有取景法即可，防止被误读成产品版面。

## G11 · 「已核对」行的绿线使用超出 signature-line 状态表字面

- **缺漏**：signature-line §二绿线 = `gate.confirmed`；矩阵/时间线里「已核对」（人核对过、非门禁确认）语义上是核对事实，字面不在五态表内。
- **临时处置**：已核对行沿用 authority 绿线 + 「已核对」chip（垂类/招投标 Q1/Q5、HR 时间线）——把「核对留痕」视为 confirmed 的同族处置态。
- **建议补册**：状态表补一行注：check 动词的「已核对」消费 authority 绿（与 gate.confirmed 同色相同语义族），或明确改为无线 + chip。二者择一拍板。
