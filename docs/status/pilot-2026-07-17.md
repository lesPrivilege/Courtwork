## 首轮发现（四项）

### A. Chat 附件正文真机未入请求 【P0 · 疑似不变量 4 违规】

现象：chat 可对话，附件 chip 正常渲染（含「仅本条」标记、needs_ocr/不支持类型 fail-closed 提示均在），但模型回复「你还没有上传任何文件或文档」——附件正文未进请求，且 UI 无任何失败提示（静默丢弃）。截图证据：两轮追问均复现，Input token 241/327 与附带多份 md 的体量明显不符。
处置：采纳，P0。`CHAT-MATERIAL-1` 的 e2e 在浏览器/DEV 链全绿，真机 Tauri 文件读取路径为独立疑点——**声称与真机不符**，先真机复现红证、查根因，静默丢弃必须转为显式失败态。
归属：`PILOT-LIVE-1` A 项。

### B. Work 场景真机未接通 【P0】

现象：Work 面可建案、场景按钮（起草答辩状/更多）渲染，但场景未能运行；无显式「未装配/未开通」反馈。
处置：采纳，P0。`WORK-LIVE-1` 的 `rejected/not_configured` 中性反馈路径在 e2e 有证，真机若连反馈都不出现则为新缺口；查真机 composition root 装配链（provider key→turn engine→work runtime→宿主 CAS）断在哪一环，接通或如实显式。
归属：`PILOT-LIVE-1` B 项。

### C. 材料入口范式错位 【P0 · 交互契约】

现象：选中文件夹后被摊平成逐文件 chat 附件上传（且未成功），而非「授权文件夹 → 案根 → 材料就地入库 → 场景消费」的 agent 范式。产品负责人定性：**这不是 agent 的范式，harness 没有实现**（指真机 live 路径未走 harness 装配，非 harness 本体缺失——工程面 258/261 有证）。
处置：采纳，P0。文件夹入口必须路由到 grant→case-root→material-ingress 链；chat 附件仅承担单文件轻量用途，文件夹拖入 chat 应引导去建案/入库而非爆平。
归属：`PILOT-LIVE-1` C 项。

### D. 布局居中不对齐 【P1】

现象：两侧栏分别收敛时 chat flow 未居中；各界面（welcome/chat/work/新案空态）的内容居中基准不一致。
处置：采纳，P1。`LAYOUT-CONVERGE-1` 已定 `--content-measure` 单源，本项为其覆盖面扩展：所有主内容面在任意收栏组合下共用同一居中基准，断言按宽度+中心点双证。
归属：`PILOT-LIVE-1` D 项。
