# 排版与密度规范（设计语言包 2/6，2026-07-09）

适用范围：全部 UI。数值以 `tokens.json` 为唯一来源，本文只讲用法与边界。

## 一、中西文混排

1. **字栈顺序即混排策略**：`-apple-system / Segoe UI` 在前承接拉丁与数字，`PingFang SC / MiSans / Microsoft YaHei` 承接中文。不手动在中西文之间敲空格，不使用 JS 加间隙——依赖字体本身的字形间距。
2. **数字与拉丁编号一律等宽化**：正文里的金额、日期、法条编号、案号保持 sans 字族但强制 `font-variant-numeric: tabular-nums`（硬性）；独立成列或成组的编号（卡片徽章、citation、文件编号、时间线日期列）直接用 mono 字族。判断标准：**要对齐的用 mono，嵌在句子里的用 tabular-nums**。
3. **中文标题字距**：≥16px 的中文标题加 `letter-spacing: -0.01em`；14–15px 正文字距为 0，不收不放。
4. **字重**：强调用 510（变量字体）或 500（回退），禁用 600/700。中文粗体在中低分屏上发糊，宁可用颜色层级（primary vs secondary）区分，不加粗。

## 二、justify 使用边界

- **允许**：仅文书阅读面（修订预览的 docx 纸面、起草画布正文）使用 `text-align: justify; text-justify: inter-ideograph;`，并同时设 `line-break: strict`。理由：这两处模拟正式文书版面，两端对齐是文书心智的一部分。
- **禁止**：一切 UI 表面——对话流、卡片描述、列表、表格单元格、批注气泡——一律左对齐 ragged right。中文 justify 在窄容器里会拉出难看的字间距河流，UI 容器宽度不可控，禁。
- 卡片与列表描述加 `text-wrap: pretty`（渐进增强，不支持则自然回退）。

## 三、密度阶（四档，全站只有这四档）

| 档位 | 字号/行高 | 用途 |
|---|---|---|
| reading | 15px / 1.6 | 文书正文、修订预览、长描述阅读 |
| body | 14px / 1.6 | UI 默认：对话、卡片描述、按钮 |
| dense | 13px / 1.5 | 高密度列表、表格单元格、时间线条目。紧凑数据列表行高 28–32px（含上下内边距，单行截断），列头 28px |
| meta | 12px / 1.5 | 元信息、徽章、角标、citation。**全站最小值，12px 以下禁用** |

- 目标用户 35–55 岁，Raycast 式 12px 正文是规避对象（docs/31）；正文永远在 14–15px。
- 列表行高：dense 列表单行内容行高 1.5，行与行之间用 padding 撑（上下各 6–8px），**不用 margin 叠加**，保证 1px 行分隔线（gridline）两侧留白对称。
- 多行截断：列表内描述最多 2 行（`-webkit-line-clamp: 2`），点开卡片看全文；卡片内描述不截断。

## 四、层级靠排版不靠色块

结构层级的表达优先级：字号档位 > 字重（400/510 二元）> 文字颜色（primary/secondary/tertiary 三层）> 间距。背景色块是最后手段，且只允许 `bg.surface / bg.raised / bg.hover / bg.selected` 四值。装饰性色块 = 违规。

## 五、KBD 组件规格

常驻快捷键提示（Linear 式，docs/31 采纳）。**是提示不是按钮**：不可点击，无 hover 态。

- 结构：`<kbd>` 单键一枚，组合键相邻并排，间距 3px，与说明文字间距 8px。
- 规格：mono 11px / 行高 1；padding 1px 5px；radius 4px；bg `#FFFFFF`；border 1px `#D5DBE2` 且 **border-bottom 2px**（微键帽感由加厚底边表达，不用 box-shadow——阴影归零纪律）；文字色 `text.secondary`。
- 修饰键用符号（⌘ ⌥ ⇧ ⏎ Esc），字母键大写。
- 出现位置：面板标题右缘、按钮标签右侧、空态说明底部。一个视口内同屏 KBD 提示不超过 4 组——它是常驻低语，不是快捷键速查表。

## 六、数字排印细则

- 金额：`3,800,000` 千分位 + tabular-nums；币种符号与单位（元）随正文字族。
- 日期：统一 `2024-08-17` 形态（ISO 短横），mono 或 tabular；中文语境叙述句内可用"2024年8月17日"但同列展示必须 ISO。
- 法条：正文内"第五百八十五条"保持中文数字（法条原文心智）；结构化引用（citation 批注、角标 tooltip）用《民法典》第585条式短形态，编号 tabular。
- 序号徽章：`R1` `E14` `V03` 等宽大写，位数按域内最大值对齐补零与否由域宽决定（同屏同域统一）。
