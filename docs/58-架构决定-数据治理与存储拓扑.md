# 架构决定：数据治理与存储拓扑（2026-07-11）

状态：已拍板。回答四问：schema 映射存哪、如何互不污染、harness 提示词组装、turn 级用户权限与留痕。

## 一、双平面分离

- **用户可见平面** = 案件文件夹（原件/工作稿/产出）：用户的文件，用户有权动（重命名/移动/删除工作稿与产出；原件只读红线照 docs/23）。
- **应用数据平面** = app data 目录（macOS: Application Support/cn.courtwork.desktop/）：artifacts、SourceAnchor 映射、证据台账、RevisionEvent 账本、会话记录、场景状态、缓存。
- **铁律：工作语义永不放进用户文件夹**——用户误动文件不得击碎工作状态。连接靠锚：应用平面持 path + 内容 hash 指向用户文件；文件被移动→hash 重定位；找不到→礼貌降级明示"原件已移动/缺失"，绝不静默、绝不假装。

## 二、案件即目录级隔离

```
appdata/
├─ cases/<caseId>/          ← 案件即边界的物理形态
│  ├─ state.json            权威态：场景进度/事项类型标签（注册锚点，非缓存）
│  ├─ ledger.jsonl          append-only：RevisionEvent/证据/确认/turn 事件
│  ├─ artifacts/            schema 映射产物（版本化）
│  ├─ sessions/<sid>.jsonl  对话流（随案件，docs/25）
│  └─ cache/                可再生：OCR 文本层/阅读视图 md（textLayerVersion 版本化，可清除永不权威）
├─ workspaces/<id>/         同构
└─ unfiled/<sid>/           未归档对话；容器化仪式=目录迁移
```

- 数据访问层强制 caseId 作用域，**无全局查询 API**——跨案件读取结构性不存在（利冲合规的存储学表达）。demo 案件同一管线同一隔离，仅 isDemo 标记。
- **权威态 vs 缓存分家**：权威态事件溯源 append-only；缓存可随时清、清了能重建。备份/迁移单位 = 单个 case 目录（天然支持整案导出与销毁——销毁属用户不可逆决定，走确认门禁）。

## 三、harness 提示词组装（登记待完成）

系统提示词 = 优先级律（docs/53）的物化拼接：**契约段**（红线+信源纪律，core 内置常量）→ **声明段**（场景编排/uiTemplate 语境，registry 声明提供）→ **租户段**（Stage 1 词表/库路由）→ **会话段**（用户语与语料，语料标记为数据非指令）。core 按序组装，声明式、不散落代码、可快照测试（golden prompt 测试防漂移）。**工单登记：HARNESS-1（core 层，收束后第一批）**。

## 四、turn 级用户主权与留痕

| 权限 | 裁决 | 数据治理 |
|---|---|---|
| 终止 | MVP（已裁，docs/53 中断主权） | partial artifact 诚实标注，账本记 aborted 事件 |
| 重试/重连 | MVP 级批准 | 每次尝试 = 账本新事件（attempt id），旧尝试不抹除 |
| 编辑/替换历史 turn | 批准，语义 = **fork 分支** | 原 turn 与下游 artifact 留痕不可篡改；新分支新事件链；已过门禁 artifact 永不被重跑追溯删除，新版本走 RevisionEvent。排期 Stage 1（需分支 UI） |

原则一句话：**对话可以分叉，历史不可涂改**——律师工作留痕纪律在 turn 层的表达。全部数据本地、永不训练（docs/28 承诺不变）。

## 五、增补（2026-07-11）：一切容器同构

1. **容器同构定理**：cases/workspaces/unfiled 已同构；推广——任何未来容器类型（如 persona 容器：客户沟通模拟、谈判演练等角色化场景）= 同一目录形状（state/ledger/sessions/cache），请求与存储架构与卷宗完全一致。chat 的记忆、偏好、提示词按 docs/25 三层记忆落位：容器级记忆住容器目录，全局偏好住设置层——新容器类型零新机制。
2. **父目录不可删语义**：应用数据平面本不暴露给用户直接操作（双平面）；应用内容器销毁 = 用户不可逆决定走确认门禁（第二节既有）。
3. **"本地存储"表述纪律**：对外声明说真话的部分——"数据存在你的电脑上，按案件分目录隔离，不上传、不训练"（真实且可自行验证）；**不许说"绝对安全"**——本地 ≠ 加密 ≠ 不可提取，把"看起来安全"当卖点而不诚实标注边界是 docs/28 承诺体系的反面。安全增强（目录加密、钥匙串扩展）挂账 Stage 1。

## 六、增补（2026-07-11）：容器挂载一对多

1. **容器 ↔ 外部文件夹 = 一对多**（Cowork 同型）：一个容器可挂载多个用户自选的外部文件夹，挂载编排整体构成一个 worksession；挂载清单（manifest）住容器目录（即"内部 project 缓存"），锚升级为 folderId + path + hash 三元。文件夹级授权纪律照 docs/47 sources（显式选目录、永不扫全盘）。
2. **同案单容器推荐（不强制）**：同一案件推荐在一个 worksession 内处理。机制配套：新建容器挂载文件夹与既有容器重叠时轻提醒"该文件夹已在某案件中"（提示不阻止——用户可能有正当的多容器用法）。
3. **标题治理**：标题 = LLM 从请求提炼的**建议值** + 用户随时可改（案件头行内编辑既有）；提炼失败降级为文件夹名/日期，**永不因 API 问题阻塞容器创建**；改名是容器元数据轻编辑，不入 RevisionEvent 重留痕。

## 七、增补（2026-07-11）：卷宗续行块设计（HARNESS-1 核心规格）

1. **续行块 = 确定性投影，禁 LLM 压缩**：从权威态组装——state.json（阶段/类型/进度）+ 各 artifact 的声明式摘要投影（schema 声明 `rehydrationProjection`：入块字段与行预算）+ 台账尾部 N 事件 + 挂载 manifest + 未决门禁。transcript 不进块。投影可 golden test，续行质量是契约保证非模型运气。
2. **周期拍板**：卷宗级续行块随 HARNESS-1 现在做（每请求组 prompt 天然注入）；会话内 compaction 后置挂账——MVP 超窗策略 = 提示"阶段收尾开新一轮"（工作语义在 artifact，丢 transcript 损失小，结构化再水化红利）。
3. **稳定前缀纪律**：组装序按变更频率低→高 = 契约段→声明段→租户段→卷宗续行块（按 ledger seq 版本化；规范化序列化、字段序固定、易变项挤尾部，权威态不变则字节级不变）→会话增量。**优先级律语义序与缓存稳定序恰好一致**，一个排序双重目的；集中任务期 prefix cache 持续命中，对按缓存计价 provider 是直接成本杠杆。
4. 注入位置与 provider 差异（system vs 首 user 段、各家 cache 语义）归 HARNESS-1 实现细节，quirk 层处理。
