# 论点：垂类 MVP 的结构优势（2026-07-10，战略终稿）

状态：经验归档（实证：意见稿到 core MVP 两天，五层验收、eval 转正、Tauri 成品）。

## 论点

1. **垂类 MVP 反而好做**：不卷通用能力（那是被上游持续打击的赛道），任务面被 harness 锁定 → 甜点区模型够用 → 无能力军备竞赛；难的部分（数据、场景 schema、纪律）不需要前沿研发，需要领域判断——团队真实经验可直接编译，上游无法商品化。
2. **核心区分点 = 场景识别硬规则 + 专属看板，且"通用不做"是激励问题而非能力问题**：有主见、维护密集、按垂类单独结账的苦活与平台玩家的横向规模激励天然错位。激励错位比能力差距更硬——能力差距被下一代模型抹平，激励错位不会。
3. **数据与场景判断线性复利 + 向上游投递**（docs/22 三资产）：上游谈接口合作更容易——我们是其 token 进入合规敏感市场的分发渠道，且回投评测与脱敏反馈。
4. **暴力碾压的蚕食诚实承认，墙立在机构层**：高认知用户用 frontier work agent DIY 会发生；但机构采购的是可培训、可标准化、可审计、有数据承诺的秩序，个体渗漏 ≠ 机构替代；渗漏者是场景挖掘的免费情报源。自足、防呆、缺口句句实话的 agent 卖的不是能力，是秩序。
5. **薄集成哲学**：task pane 薄客户端、通道网关适配器、邮件源适配器——最小实现接入各 workflow 比重集成更可靠、维护面更小（与"厚薄算法"同源：边缘做薄、判断做厚）。

## 廉价模型时代的推论（2026-07-10 增补，外部信号：Meta Muse Spark 1.1 / 腾讯混元 Hy3——上代前沿水平、约 1/10 价）

1. **该级别模型是甜点区机制的目标客户**：有界任务 + 固定编排 + schema 校验 + strict 降级链，正是"上代水平模型"的可靠性放大器——在本架构里它们不需要打过前沿，够用即最优性价比。混元类有 OpenAI 兼容端点即进 eval 候选池（core SPEC 已记）。
2. **定律：模型越便宜，编排的价值越高**——自由发挥把便宜 token 变成便宜 slop（弱模型自由态失败率更高、每次随机画 HTML 铺最后一公里），硬编排把便宜 token 变成可靠工作。冰山下 90% 的需求要的是可靠性下限而非智能上限，可靠性下限由 harness 保证。"卖秩序"在廉价模型时代增值。
3. **生态玩家是通道不是对手**：腾讯的通用微信 agent 不会做卷宗朱批（激励错位），但教育了"IM 里有 agent"的用户心智——通道网关（企微适配器，Stage 1）搭车不撞车。其"模型 6 个月可追、生态 10 年难建"的时间错配论，即本仓库"基座可换、数据/场景/专家门禁换不掉"的宏观版。

## 终章推论：场景生态的生产函数（2026-07-10）

coding agent 是铺垂类最后一公里的工厂机器（本仓库实证：五层 harness + 四场景两天铺完），架构宪法是质检，专家判断是唯一稀缺原料，场景注册表是成品货架。垂类边际成本坍缩至"数据 + 判断"两项。三重最优的机理：**经济**——schema/工具编排是 coding agent 量产的一次性资本品，复用零边际；**幻觉控制**——schema 校验 + 确定性接口是结构性幻觉牢笼，coding agent 使"每垂类一套牢笼"便宜化，幻觉治理从提示词工程变为基建工程；**交互**——固定场景 UI 的可预期性完胜逐轮生成 HTML。生态终局：场景声明是数据、可交换可发布，入场规则已定（docs/24 产出 schema 与门禁必填），场景货架可对外开放——律所共享审查场景、协会发布合规场景、第三方为自己的垂类铺场景，同跑一个"模型只生成、不裁决"的 harness。

## 一句话版

通用 agent 卖能力，能力在贬值；垂类 agent 卖秩序（场景、防呆、承诺、留痕），秩序在复利。**模型越便宜，秩序越值钱。**
