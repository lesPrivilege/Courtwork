# 长期 Roadmap：法律垂类 Agent（内部对齐口径，2026-07-09）

承接《MVP 规划》。Roadmap 不按时间排，按**条件触发**排——每个阶段有明确的进入条件（上一阶段的验证指标达标）和退出交付物。北极星不变：当法律从业者进入某个团队时，"该团队是否配备本产品"成为真实考虑因素。路径逻辑：**高频 → 沉淀 → 独占数据面 → 价值性质从"更快"变成"不可缺"**。

## Stage 0 — MVP（见 MVP 规划）

退出条件：试点团队主动摄取真实卷宗；场景按钮使用占比过半；修订采纳率形成基线；至少一家试点表达续费意向。

## Stage 1 — 场景扩张与入口下沉

进入条件：Stage 0 达标。核心命题：**从"一个好用的工作台"变成"长在用户既有工作流里"。**

- **场景注册表机制化**：从硬编码四场景切换到注册表驱动，产品团队不动核心即可周级上新。场景扩张的优先序来自驻场 + 遥测的挖掘机制（Stage 0 已建），而非内部想象。候选池按调研分类树推进：庭前准备（出庭提纲/庭审问题）、证人笔录分析、尽调报告、法律意见书/律师函、定义项与交叉引用检查（国内空白）。
- **WPS/Word 插件入口**：调研确认海外产品默认长在 Word 里、国内插件生态是弱项。修订/批注/条款面板下沉到文档内，桌面 app 保留旗舰体验（卷宗、矩阵、图谱），形成"重活在工作台、轻活在文档里"的双入口。
- **轻对话模式（chatbot 合并，2026-07-09 增补）**：不绑案件的全局对话 session（法律问答、快速咨询），与案件工作台同一 core、同一客户端，差异仅在 system prompt 与 memory 策略——参照 ChatGPT/Codex 客户端合并趋势。守门：不产 artifact、不豁免信源纪律（引法条走 cite-check 标注、生成与确定视觉区隔照旧）。
- **检索与引用底座加深**：接通类案/法规库的引用溯源与校验（cite-check 范式），推进官方/准官方数据源合作落地——这是对自建库玩家（法宝/得理/Alpha）的差异化正面战场，宜早不宜迟。
- **eval 集 v1 成型**：试点真实任务沉淀为评测集，接入模型选型流程，甜点区模型每代重选机制开始运转。
- **IM/工作流通道网关（符合国情的移动面，通道无关）**：不做独立移动 App。core 事件流协议外挂一个**通道网关**，通道作为适配器接入：企业微信、飞书、钉钉、律所内部 OA/workflow 均为同一抽象下的实现（官方 API 路线；个人微信 hook 方案因 ToS 与合规风险明确排除）。统一交互模型：摄取完成/确认节点/告警推送到通道 → 用户在消息卡片上确认或驳回 → 桌面工作续行；绑定走桌面端扫码/组织授权；富交互（时间线/图谱查看）由通道内小程序/网页承载。**红线（对所有通道一致）：卷宗内容不出通道商服务器可控范围，推送只含脱敏摘要 + 深链，全文经鉴权查看或引导回桌面**；私有化客户用其自有 IM/OA 的自建应用保持数据在租户内。Stage 3 监控告警复用同一网关，成为"自主 routine 助手"的载体（口径不变：推事实、留判断）。选择哪个通道先做，由试点客户的实际 IM 分布决定，不预设。

退出条件：非试点客户的自然增长出现；插件端日活占比可观测；场景库达到"采购演示清单"的完整度（覆盖目标客户 80% 日常法律事务）。

## Stage 2 — 数据复利与团队化

进入条件：Stage 1 达标。核心命题：**个人习惯升级为团队资产，复利飞轮（线性的）开始转。**

- **反馈标注 → 精调闭环**：Stage 0 起积累的 schema 级修正（时间线修正、风险点否决、修订保留/拒绝）达到可训练规模，启动第一轮垂类精调；同时反哺场景识别分类器与检索校准。口径提醒：这是线性增强不是指数飞轮，靠专业团队持续人工校准——而"没有飞轮"正是在位者壁垒成立的前提（意见稿既定判断）。
- **团队协作层**：一个案件 = 一个共享文件夹 = 共享 AI 上下文 + 分工批注。调研确认"文件夹级多人协作 + AI"无标杆产品（Legora Portal / Lexis Workrooms 刚起步，iCourt 偏项目管理）——这是从个人工具变成团队标配的关键一跃，也是席位数自然扩张的机制。
- **主体图谱升级为团队级持久资产**：当事人/企业图谱从单案产物变为跨案件、跨合同复用的图谱库。这是 Stage 3 全部监控与穿透分析场景的地基，必须在本阶段完成资产化。
- **机构知识沉淀（firm knowledge）**：历史案卷批量摄取 + 本所文书先例索引，对标 Harvey Knowledge / DraftWise 的 DMS 索引模式；配合"摄取即锁定"的商业动作。

退出条件：团队级功能的席位渗透率（同一客户内多人日活）；图谱库覆盖客户主要常年业务主体；精调模型在 eval 集上对通用甜点区模型形成稳定优势。

## Stage 3 — 资产监控层（价值性质切换）

进入条件：图谱资产化完成 + 官方数据接口落地。核心命题：**从"用了更快"到"不用就有损失"。**

- **合同履约监控**：履约节点提醒、义务跟踪、续约与成本机会挖掘（Ironclad 2026 三 agent 是方向参照，国内空白）。
- **对手方/关联方风险推送**：基于团队图谱库，对手方工商变更、涉诉、被执行等变动自动推送——数据订阅连接器框架（规划稿预留结构位）在此启用。
- **投资主体穿透分析**：母公司/投资主体的诉讼风险、履约风险穿透报告，服务企业法务与投融资场景。
- **法规更新预警**：客户业务画像 × 法规/司法解释变动的定向预警（调研确认国内 AI 化预警未成气候）。
- **交付与计费**：监控产出是"事实推送 + 留人判断"的报告/告警，按主体/合同数订阅计费（Plan 附加包）。**口径纪律：这一层对外叙事严守协作者边界，是持续推送事实的哨兵，不是自动决策的数字员工**——它恰好是既能做重、又不越判断权边界的自主形态。

退出条件：监控订阅续费率（这是全产品粘性最强的收入）；客户把监控告警纳入内部合规/风控流程（= 进入机构制度，拔不出来）。

## Stage 4 — 标配化与生态

进入条件：Stage 2–3 的沉淀与监控在头部客户跑通。核心命题：**从产品变成行业基础设施。**

- **被集成**：开放 API / 插件框架，接入案管系统、邮箱、企业内部工作流（通义法睿的"百炼 API 被集成"定位可参照，但我们输出的是场景与图谱能力，不是裸模型）。
- **行业 playbook 与场景生态**：预置 playbook 按行业线扩张（金融、地产、制造……），可引入外部专业机构共建——专业团队维护数据的能力对外产品化。
- **新人即用**：入职新成员继承团队场景配置、图谱与知识资产——北极星的字面兑现。招聘市场信号（从业者主动询问团队是否配备）作为最终验收指标观测。

## 数据资产里程碑（贯穿各 Stage 的暗线）

- Stage 0：eval 集 v0；反馈捕获机制上线（标注量从 0 到有）。
- Stage 1：eval 集 v1 进选型流程；官方数据接口首批落地。
- Stage 2：标注达精调规模；团队图谱库资产化；机构历史案卷入库。
- Stage 3：数据订阅流常态化；监控规则库（专业团队维护）成为新的数据资产类目。
- Stage 4：行业 playbook 库对外生态化。

模型、框架、UI 壳在任何 Stage 都可换；这条暗线上的东西换不掉——它就是意见稿说的护城河的工程展开。

## 风险与对冲（按 Stage）

- **S0–1｜红海挤压**：合同审查与类案检索已卷 → 不做单点主打，靠摄取管线 + 矩阵审阅 + 修订交付的组合差异化；Robin AI 被微软整体雇走、Spellbook 涨价是单点工具脆弱性的直接警示。
- **S1｜官方接口政策依赖**：独占合作既是壁垒也是单点 → 数据源多元化预留，接口层与场景层解耦以便替换。
- **S2｜复利不及预期**：标注量或质量不足 → 反馈捕获是 UI 设计问题而非运营问题，schema 级修正的交互纪律从 Stage 0 就不许破。
- **S3｜过度承诺**：监控/自主类场景最易在宣传中越界 → 对外口径审查机制，严守"事实推送 + 留人判断"。
- **全程｜上游模型冲击**：通用 agent 能力持续上探 → 甜点区 + eval 集机制保证"模型变强我变强"；壁垒从不押在模型层（意见稿既定判断，不重复论证）。

## 总结

五个阶段一句话：MVP 让用户把卷宗交进来；Stage 1 长进工作流；Stage 2 把个人习惯变成团队资产；Stage 3 把价值从"更快"变成"不可缺"；Stage 4 变成行业基础设施。阶段之间条件触发、不许跳环——尤其不能在高频未成立时提前推资产叙事，不能在图谱未资产化时提前上监控。
