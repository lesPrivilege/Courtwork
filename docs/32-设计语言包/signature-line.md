# 「法理之线」交互规格（设计语言包 3/6，2026-07-09）

签名动作，全站唯一的非常规元素（docs/31 拍板）。隐喻：纸质卷宗侧边朱批。它是**处置状态的物理载体**——线体只表达当前处置状态，不混入严重度这一第二维度；无语义状态时不渲染线体。

## 一、解剖

```
┌──────────────────────────────┐
│▌R3                            │  ▌ = 线体：2px 竖线，贴卡片左缘，通卡片全高
│▌ 验收标准模糊且未约定书面异…  │  R3 = 徽章：线体顶端右侧，12px mono，
│▌ 依据：《民法典》第621条 [A]  │       与卡片首行文字基线对齐
│▌ [待确认] 确认 驳回 修正      │  [A] = 信源角标：独立元素，永不上线体
└──────────────────────────────┘
```

- 线体：宽 2px（`borderWidth.line`），高度 = 卡片内容全高，位置 `left: 0`，卡片自身 padding-left ≥ 14px 给线让位。
- 徽章：12px mono 大写「域字母+序号」，色相跟随线体，但文字取对应 `semantic.*.fg`，不直接复用图形色。
- 卡片其余部分照常：1px hairline 描边、radius md、无影。

## 二、状态表

| 处置态 | 线色 token | 何时 |
|---|---|---|
| 高危待处理 | `line.danger` #DC2626 | `gate.pending` 且 `severity.high` |
| 依据未核验 | `line.attention` #D97706 | 当前依据含未核验源，严重度不限；与高危待处理并存时红色优先 |
| 已修订未确认 | `line.revision` #2563EB | 已提交修订、尚未确认最终处置 |
| 已确认 | `line.authority` #16A34A | `gate.confirmed`，严重度不限 |
| 已驳回 | `line.neutral` #64748B | `gate.rejected`，严重度不限；灰表示退出工作集，不表示错误 |

`severity.medium` / `severity.low` 的待处理项均**无线**，严重度只由等级列徽章表达；普通卡靠 1px 中性边框或背景色差分层。五类线色均为 `tokens.json` 的图形色，只用于 2px 线体与图形。徽章、状态和 citation 文字必须消费对应 `semantic.*.fg`（红 `#B91C1C` / 琥珀 `#B45309` / 蓝 `#1D4ED8` / 绿 `#15803D` / 灰 `#475569`）；其他文字色仍按 token 用途消费，不因线体状态表删减而失效。

## 三、多状态并存的优先级

一张卡片只有一条线、一个颜色。并存时取序最高者：

**danger > attention > revision > authority > neutral**

优先级用于同一时点确有多个状态信号时的消歧，不把严重度重新混入已落定状态。示例：高危待处理且依据未核验 → 红；中危已确认 → 绿；高危已确认 → 绿；任何严重度已驳回 → 灰；中/低危待处理且依据已核验 → 无线。

## 四、徽章字母表

| 字母 | 域 | 示例 |
|---|---|---|
| R | 风险点 | R1–R6 |
| V | 修订指令 | V01 |
| E | 时间线事件 | E14 |
| Q | 审阅要素（矩阵列问题） | Q3 |
| D | 卷宗文书 | D04 |
| P | 主体/当事人 | P2 |

- 序号 tabular mono，同屏同域位数统一。
- **A/B/C 永久保留给信源角标**，禁止用作徽章域字母，避免视觉撞车。

## 五、与信源角标的关系

- 信源角标（16px 方格、12px mono 单字母 A/B/C，`semantic.tier.*` 配色）挂在**每条依据/引用**上，是引用级属性。
- 法理之线是**卡片级处置状态**。信源等级只有一项联动：当前依据含未核验源时进入 attention 琥珀；全部依据已核验不会自动产生绿色线，只有 `gate.confirmed` 才进入 authority 绿。
- 角标点击 = 溯源跳转（原文定位高亮）；线体本身不可点击。

## 六、hover 折光与 citation 浮现（docs/31 二次裁定吸收）

- 仅悬停**展开详情卡**时：线体在**鼠标 Y 坐标**处向两端折射一段约 40px 的冷光渐变（当前线色与透明度叠加，不现场发明提亮 hex，`hoverGlowLength: 40`），随鼠标垂直移动跟随；渐显 120ms ease-out（`motion.hover`），移出即消（0ms）。
- 同时在卡片右缘外侧浮现 12px mono 的 citation 批注（该卡第一条依据的短引，如 `民法典 §585`），仅 hover 存在，不占布局。
- 折光与浮现是**悬停反馈**，属于允许动效；**状态变更（线换色）永远 0ms 硬切**，两者不冲突、不混用。触屏环境无 hover：折光与浮现直接不存在，不做替代交互。

## 七、动效纪律

- 线色切换：`transition: none`。确认一个风险点，红→红（见优先级）或琥珀→绿是**瞬间**发生的——硬切本身就是"落章"的仪式感。
- 线体不参与长任务动效（微光流转发生在卡片 1px 描边上，不在 2px 线体上，见 principles.md）。
- 数据区静止原则同样约束线体：无呼吸、无脉冲、无渐变循环。

## 八、退化与边界

- 卡片高度 < 40px（单行密集列表）：仅当该行具备上述五类处置状态时保留线体，徽章省略；无语义行无线，只做 120ms ease-out 中性 hover 底，不做折光与 citation 浮现。
- 嵌套卡片：只有最外层语义容器有线；内层依据条目用角标不用线。
- 打印/导出：线体照常输出（2px 实色），折光与浮现不存在。

## 架构修订（2026-07-10，拍板补录）

**静默态取消**：原规格"所有内容卡片左侧常驻冷灰竖线"废止。无语义状态的卡片无左线；法理之线只在语义态出现（2px 硬切，红/琥珀/蓝/绿/灰），出现即有含义。理由：常驻标记不携带信息，且"每卡一线"是 AI slop 观感的直接来源（docs/35 de-slop 清单第 1 条）。本次已同步清理状态表、优先级和密集行边界，正文不再保留冲突表述。

**处置状态单维收敛（2026-07-10，P-1）**：严重度保留在等级列徽章，线体不再把中/低危映射为琥珀/灰。高危仅在待处理时为红；已确认不论严重度均为绿；已驳回不论严重度均为灰。依据未核验仍以琥珀覆盖任何严重度，优先级为红 > 琥珀 > 蓝 > 绿 > 灰。
