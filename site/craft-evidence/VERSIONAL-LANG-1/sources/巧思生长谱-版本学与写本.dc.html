巧思生长谱 · 从文本载体传统替换路径依赖

要用版刻与写本传统替换 Claude Design 的常见套路（左侧彩色衬线条、01/02/03、玻璃卡、uppercase kicker、渐变 CTA），关键不是「换一套皮肤」，而是一台可复用的生长引擎：陌生化的本体是编排的多样性，不是字体的猎奇。记号走版本学（界行 / 鱼尾 / 版框），字形走写本正脉（书手谱系）；表达位取写本源、功能位取刊本源；色墨守配额（朱仅裁决 · 泥金唯一强调）。

生长法则 · 一条新巧思的准入五问

答不出任一问即不长出——这是把「有品味 = 禁止清单」升级为「有出处 + 有职能 + 有门禁」。
一

有出处

源自文本载体传统——版本学（版框/界行/鱼尾/象鼻/牌记/校勘）、写本经书（书手/朱笔/抬头/题记）、装帧（经折/卷轴/线装）。不是视觉猎奇（仿哥特、Papyrus、novelty 展示字）。

二

有职能

编码一项真实语义功能，并指名它替代的一个具体 slop trope。无对应职能的记号即违例——「掐边花纹一律拒」的推广。

三

从 token 构造

零新色值、零新阴影、零新圆角。记号只用 currentColor + 既有线级（rule.major/minor）与间距阶。几何单源（一份 <symbol>，处处 <use>）。

四

绑档位与门

绑定唯一激进度档（激进 / 中间 / 克制）+ 一条机器门；许可不跨档继承。激进档探得的成熟项须过中间档克制审计才回迁工作面。

五

守分轨

记号色古（朱/红/琥珀承文书传统）、交互色今（蓝绿）；表达位写本、功能位刊本；数据区静止，朱只落在人做决定处。

回迁护栏（效果图是探索件，不是权威）

  • 效果图只探组合与布局；被采纳须翻译为 token / 组件 / blueprint 才生效。
  • 回迁唯一通货是既有 token 与设计语言——三层表面、动效四属性白名单、零投影密度对候选同样有裁决权；新色/新阴影/弹簧动效一律回迁时剔除或先修 token。
  • 外部参考（含本 kit 的冷调站）经源码级调研入档才可作依据，视觉皮肤不随机制迁入。

一句话

替换路径依赖不靠「换字体」，靠一台生长引擎：从文本载体传统取记号、为每条记号绑定真实职能与机器门、只用既有 token 构造、守记号色古/交互色今与档位不跨继承。本谱自身即刻本印页浅宗、文武线主界、数据区零动效、mono 承数字与门名。
