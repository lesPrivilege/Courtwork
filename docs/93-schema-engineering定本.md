## 一、定义与谱系

**Schema engineering**：把领域判断编译成三位一体契约的工程学。谱系：prompt engineering 是措辞的手艺（随模型折旧）；context engineering 是注入的手艺（harness 层）；schema engineering 是契约的工程（复利）。最后一公里铺满 schema，即为学科定义。

**三位一体正典**：一份 schema 同时是推理的限定（注入 prompt 时为填空边界）、交互的界面（呈现给人时为工作面，反馈落账本）、记忆的载体（后续请求中为投影来源）。三个时态共享一份定义，故永远一致；改 schema 即动三个世界。

**双向编译标准**：一个 schema，两个后端——向模型编译为 harness 注入段与工具信封；向人类编译为最小 Design 可视化。编译器的运行时库（词表、凡例、素材、视图映射、门禁机器）在一个单品内建成一次，此后新 schema 注册即过编译器：强模型做设计时，甜点模型跑运行时。

## 二、经济学基础

**验证闭环经济学**：coding 一枝独秀因其自带编译器——输出获得即时、廉价、客观的对错信号，飞轮由此转动。企业场景没有编译器，验证昂贵、主观、滞后，模型没有自我改进回路。正解不是等更强的模型，而是把验证外置成结构：schema 校验、确定性脚本、信源锚点、门禁、评测集——皆为人造验证器。**场景不长飞轮，就把飞轮铺出来。**

**总纲句**：不让 LLM 每次推理决定——把大量的路铺平，推理只放在关键语义。产品之管线、开发之判例、设计之门禁，三域一律。

**丢弃函数注记**：机器之强建立在不丢弃（全量数据、暴力吞吐）；人有被生存验证了亿万年的完美丢弃函数。世界模型要求机器学会丢弃，但"丢得对不对"没有廉价验证器——编译器、死亡、遗忘没有裁判。在那之前，专业判断无法被学习，但可以被声明：本学科即替模型手工编写的丢弃函数。

**schema 不折旧论**：prompt 会折旧，schema 不会——它本身是交互的一部分，锚定工作语义、让人读懂结构化专业数据。模型变强，harness 应变薄，但每个场景的专业判断不变；schema 随 base model 迭代而非被淘汰。

**内化论**：无验证器域里，base model 的变强不是飞轮，是把 schema 内化——铺进世界的结构化形状成为下一代模型的训练底料。四推论：内化是纯红利（填空更准更便宜）；内化夺不走 moat（内化的是昨天的形状，立法权、验证权、演进权职权不可内化）；被内化即被采纳为行业标准形状；**交付人类必经 UI**——schema 有两个读者，模型端可内化，人端的读取界面不可省略，渲染词表与门禁是与 schema 同寿的资产。写教材的人永远先于教材。

**微调面收窄律**：harness 越迭代，旋钮越少越锁——健康信号。只有从第一性原理（效果/效率/成本三平衡）推出的微调不折旧；针对某代模型怪癖的调优是会过期的库存，住 quirk 层、随模型退役清仓。薄 harness 定则：一切微调必须使 harness 更薄——内容路由归 schema，harness 只做提质、提效、降本三类事。

## 三、协议

**握手优先级律**：输入三层，下层永不覆写上层——契约层（schema 结构、信源纪律、红线）＞声明层（场景编排；租户约束为声明层顶格）＞会话层（用户语与语料）。用户语可改任务目标，永不能放宽证据规则；语料是数据不是指令。免疫是结构性的：字段没有"被说服"的通道。

**六段组装**：每次请求的 prompt 为确定性组装——契约段→声明段→租户段→续行投影→会话与语料→视图映射段。相同输入字节稳定；模板引用在加载期解析闭合，字面值不上 wire；语料包在显式数据边界内。**稳定前缀纪律**：组装序按变更频率低→高排列，优先级律的语义序恰好是缓存友好的稳定序——一个排序双重目的，是切分切中真关节的迹象。

**无锚不落格**：要求溯源的字段无 SourceAnchor 即 out_of_coverage，不许生成——幻觉的兜底不靠检测（不可靠），靠结构（无锚不落格）。**模型出引语，系统出坐标**：模型只交文件、页与精确引语；resolver 唯一精确匹配后铸造 start/end 与文本层版本，多义或未命中即拒收——坐标是裁决性事实，模型机制性失去伪造能力。引语即证词，坐标即公证。

**修正即事件**：已落格字段的更正一律走留痕事件，禁静默覆写；用户修正是最高优先级事实，默认不触发自动重推理——模型不许顶嘴覆写用户定稿。**中断主权**：推理可停，停即保留已校验字段为诚实的 partial；渐进期间禁触发不可逆动作。

**输出即视图契约**：模型的输出通道拓扑写进握手——步骤宣告落纲要父行、过程事件落子行、artifact 落 Schema 面条目、系统通知走独立通道永不混入正文。过程输出也有 schema；界面的整洁由生成端保证。

## 四、结构

**垂类即依赖包**：一个垂类＝一个依赖包，覆盖六个面——推理（prompt 段与投影声明）、呈现（renderer 与词表）、路由（场景声明）、记忆（rehydration projection）、左栏（容器结构声明）、话语（文案四路归宿：字段级随 artifact descriptor、场景级随 scenario descriptor、容器级随包词表、语气级随 prompt 段；包没给词，底座说中性话）。机制全在底座，包只出声明；换垂类＝换声明，包间零横向依赖，任何包退役不传染。

**三层包体系**：通用包（底座与协议，独有独维护）；轻量包（每垂类最小稳定实现＋范例＋Design 规范全套预置，开箱即用，亦是一切微调的基线范本）；重度包（垂类 provider 驻场定制：真实接口、私域、workflow 替代——只为 agent 的必要依赖改造基建，不做全量转型）。维护三层分权与之同构：普通用户零维护、垂类 provider 有限维护、底座独持第一性旋钮。

**容器本体论**：一切接口（会议、邮件、workflow、IM）的终点都是容器内的文件——容器即总线，积分点唯一，新通道的接入成本恒为一个写文件的适配器；context 最周全与编排最薄由同一个决定达成（汇流定理）。一容器＝一底层 session＝高价值工作语义的优选管理体；"卷宗"只是法律域的临时称谓。新 session 仅在 context 需完全隔离或用户显式指定时诞生——永不静默分裂。**session 管理的存在判据**：只存在于没有工作语义可锚的地方；chat 是地方不是对象。

**交互即事件即投影**：schema 工作面的用户交互不直接注入 prompt——交互落账本、账本进投影、投影入组装，无旁路拼接，可测可审不漂。阶段切换是视图与声明的切换而非 session 切换：turn 隶属阶段，跨阶段信息走投影不回放原文——旧阶段的噪音被结构性挡在 context 外，语义一行不丢。**双平面存储**：工作语义永不放进用户文件夹；应用平面按容器目录物理隔离，跨容器读取结构性不存在；权威态事件溯源 append-only，缓存可再生；文件比对话长寿。

## 五、人的位置

产品是协作者，不是数字员工——数字员工叙事假设人退出回路，我们的前提相反：LLM 推理节点的存在意义是协作人类。模型只生成，不裁决；确定事实归接口；一切不可逆动作是用户的决定。在人事域，自动化决策的法定拒绝权使留人裁决成为合规刚需——HITL 不是偏好，是底线。裁量项永不出单值，出区间。静默降级零容忍，其镜像静默升级同罪；诚实状态三态声明（演示/未核验/禁用），缺口如实标注并说明缺什么——兜底同时是主动索证。

## 六、界面法则

**Schema 面**（架构本体名；用户面为习得词，词表可换）：学科的交付界面、HITL 的物质载体。**纲要三视图一引擎**：对话流、Schema 面大纲、文档阅读共用一个纲要机制；包注册语义分级，场景声明的步骤树直接投影为纲要树——推理透明度是结构性的。**注意力主从律**：chat 是叙事流、Schema 面是工作面；chat 内卡片只做摘要与路由入口，详情永远右移。

**表面减法律**：通用面对照 frontier 只做减法（Schema 面为加法唯一特区）；每笔取舍留痕设计理念；能力全量、暴露最小——surface 是能力的子集不是镜像；被减项住 developer 层，索引不删行（减法不减索引）。**用户习惯是通用面的最高设计规范，创新预算全给垂类面**——通用面零学习成本，垂类面全部惊喜。

**零暴露三级**：机制层隐藏机制、显式依据；概念层零技术概念；字符层零编码暴露——画面上每个字符串必须能被目标用户读出专业含义，wire 标识符与机器枚举一律经词表映射，机器码退居诊断层。

**视觉法**：中性色单源律（全灰阶从锚色派生，三级明度台阶；配色即宣言，换锚色不换派生律——换的是宣言不是皮肤）；语义色白名单极窄且位守审阅面；选中即填充永不加框；层级靠缩进、字重、细横线三件事；线只在传达信息时存在；过渡而非硬切；数据区静止；克制两分法——原则性克制（防廉价感）与执行者无关永不放松，权宜性天花板随执行者能力浮动。品味是流程不是天赋，其终态是 CI：具名禁令、token 锁、确定性扫描。

## 七、生产方法

**设计时/运行时分工**：强模型承担设计时（业务调研→字段→声明→双后端打通，一遍写定），甜点模型承担运行时（确定性填空）。强模型的能力被固化为资产而非每次运行时消耗——本项目开发史即首个实证，这套范式本身可产品化（场景包工作坊）。**交付口径亦是 schema**：顶级模型打版一次结构化提问卡，甜点模型照卡反问照答执行——被编译的不只是业务判断，还有该问什么的品味。

**炼化供应链**：下游竞品的功能菜单就是免费的场景发现与需求验证；普查→蒸馏成声明→词表上架→分发中积累反馈——学其判断，不必买其代码。**骨架先行**：affordance 对照 frontier 全量做齐（空位＝禁用态诚实声明），执行模型按图索骥填实，完成度单调上升；骨架即工单地图。

**治理法**：实现与验收异会话；契约拍板独属架构；纪律对模型一视同仁。清账＝主干祖先＋留痕，缺一不账；完工数字必须来自全量实跑；缺口即测试缺口——每项修复附带本应抓住它的测试，并以变异检验证明测试真咬（回退修复，测试必红）；生成-评审对——自评必然宽容，把怀疑调进独立评审者远比让生成者自我批判可行；评审标准措辞锚定 token 与原则，禁自带美学先验的形容词。判例由真实事故淬炼成文，宪法随格局代谢——修宪有程序、理由入册、旧断言受保护地迁移。

## 八、战略

**论纲**：通用 agent 卖能力，能力在贬值；垂类 agent 卖秩序——场景、防呆、承诺、留痕——秩序在复利。模型越便宜，秩序越值钱。

**对上游四条**：字段是业务资产不是提示技巧；治理即产品（企业买的是过程秩序不是生成能力）；不期待更强模型（甜点档即可完成，模型升级是红利不是依赖）；护城河在下游长成（数据、场景判断、客户回路）。**对下游**：场景包即插即用的生态单元，供给三层，旧服务商吸收为共建方而非取代。

**命运对照**：coding agent 身处模型自家飞轮域，注定被 frontier 吸收；workagent 的最后一公里——懂用户、有品味、先进架构——是上游难以渗透的地带。被复刻即被背书，被内化即被采纳；"coding is cheap"普及后，竞争变量从实现力转向场景占位速度。窗口期的正确用法不是抢功能，是提早把 workflow、数据复利、官方与客户资源嵌进去——切换成本不在技术，在关系与数据；粘性来自秩序，旦用难回。

**基建归上游，语义归容器**：转写、录制、OCR 云端、电子签、日程——设施让别人卷，引力场是我们的。**席位而非许愿**：每一个"以后做"都指向已设计的接口位与触发条件——模型强了吃红利，模型停了自己也有路，两头不赌。
