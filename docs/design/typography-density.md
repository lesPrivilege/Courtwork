# 排版与密度（排印凡例）

**发凡起例（2026-07-19 立）**：本文是产品壳字体、字重、选色的编排法——律先行，值表随 **B2-0 定值批**落 `tokens.json`（沿 B0 判例：原型提值、数学校值、门禁锁值、诚实标值）；争点见文末，未拍板项不得实现。数值以 `tokens.json` 为准。

## 发凡一 · 三轨字体制

| 轨 | 承载 | 字体 | 字重 |
|---|---|---|---|
| **标题轨** | 视图标题、章节题、hero 级 | 通行宋体（思源宋体类 OFL，具体字与字重阶待 B2-0） | ≥双字重梯度 |
| **文书轨** | 阅读视图正文、引语、修订预览（长阅读面） | 仿宋陌生化轨——聚珍新仿候选（许可**三张**：个人非商业 / web 嵌入 / 桌面内嵌发行，逐张核实留快照；任一未核清即以朱雀仿宋 SIL OFL 落地，不悬置不静默替换） | 单字重（零粗体律） |
| **功能轨** | UI chrome：按钮、标签、输入、导航、设置 | 系统 sans 栈**续任**（现行法），显式定性为「工具字」——不承载气质预算，气质由标题轨与文书轨承载。`[需产品拍板：拒苹方条款是否及于功能轨，见争点一]` | regular / emphasized 两档（现行法） |
| **数据字** | 编号、日期、金额、表格数字 | mono 或 `tabular-nums`（现行法不变） | — |

三轨定性即 kill-slop 的字体面完整表述：slop 判定在于**用默认字体做表达性内容**，不在于工具位用了系统字。表达性内容（标题、文书）一律出轨着装；工具位显式声明工具字身份。

## 发凡二 · 字重即层级

与线级语法（principles §10）同构：字重携层级语义，非装饰。功能轨维持两档；标题轨字重阶由 B2-0 实测定；**仿宋轨零粗体律**——仿宋无原生粗体，伪粗体一律禁（`font-synthesis: none` 入静态门）：文书面强调走记号系（圈点/侧点）、字号与墨色，不走加粗。避免 600/700 的中文糊重（现行法，续）。

## 发凡三 · 墨色律（文字选色）

正文即墨：文字只用 `text.*` 中性阶；彩色文字仅承载语义（语义色稀缺性裁定既有：红/琥珀仅风险、绿仅落定、朱仅印记）。字体×字号×字重×色**四元联测 AA**（B2 票面④：仿宋实际度量为准，不得沿黑体度量宣称）；tertiary 12px 既存缺口（三面 4.23/3.98/3.84，批次账 B2②）的闭合路径由 B2-0 在字号升档 / 值面复审 / 轨位调整间实测定谳。

## 发凡四 · 度量律

字栈与字号槽 token 化（`--font-title` / `--font-body` / `--font-ui` 类，B2-0 落 `tokens.json`）；仿宋正文字号/行高补偿（仿宋视觉偏小偏窄）；CJK–西文混排配衬字显式指定（仿宋拉丁字形弱，不得裸回退）；≥16px 中文标题可轻微负字距、正文不调字距（现行法，续）；桌面壳字体随包内嵌不走网络——子集化属体积纪律，`font-display` 属站面义务。

## 起例 · 槽位表（值待 B2-0，本表定测点）

| 槽位 | 轨 | 密度档 | 色槽 | 测点 |
|---|---|---|---|---|
| 视图标题 | 标题 | body | text.primary | 双宗 AA |
| 章节题 / 表头 | 标题 | dense | text.primary | 双宗 AA |
| 文书正文 | 文书 | reading | text.primary | AA 4.5 双宗 |
| 文书引语 | 文书 | reading | text.secondary | AA 双宗 |
| 表体文字 | 功能 | dense | text.primary | 既有 |
| 表体数字 | 数据 | dense | text.primary | 等宽 + 数据区静止 |
| 元信息 / citation | 功能 | meta | text.tertiary | **缺口闭合测点** |
| 按钮 / 输入 | 功能 | body | 现行 | 既有 |
| 批注 / 落定章旁文 | 文书 | meta | 朱仅印记 | 消费面随 B4 |

## 四档密度

| 档位 | 用途 |
|---|---|
| `reading` | 文书、修订预览、长阅读 |
| `body` | 对话、说明、按钮与普通卡片 |
| `dense` | 表格、列表、时间线与矩阵 |
| `meta` | 元信息、徽章、角标与 citation |

`meta` 是全站最小正文档位；不得把 12px 当普通正文。列表行通过 padding 形成高度，不用相邻 margin 叠加。

## 对齐

- 正式文书阅读面可以两端对齐并启用严格中文换行。
- UI、卡片、表格和对话统一左对齐，避免窄栏中文 justify 拉伸。
- 金额右对齐；同列日期统一 ISO `YYYY-MM-DD`；域编号同屏位数一致。

## KBD

KBD 是不可点击的低语提示：mono、紧凑、细描边、无 hover。一个视口内不超过四组。它不能替代可发现的按钮，也不占用主操作层级。

## 机器门清单（B2-0 随值表立门）

- **字栈单源门**：`src/` 内 `font-family` 声明必须 ∈ token 字栈声明集（仿中性色单源律形态）。
- **伪粗体门**：`font-synthesis: none` 存在锁 + 字重白名单（仿宋轨出现 >regular 字重即红）。
- **数据字门**：数据区 mono/`tabular-nums` 断言（对齐律机器化）。
- **AA 四元联测**：仿宋实际度量入 e2e（B2 票面④）。

## 争点 `[需产品拍板]`

1. **功能轨归属**：甲·系统栈续任（推荐——12px 控件可读性优先，工具字定性显式即非 slop）；乙·宋体统揽 UI（气质最重，小字号可读性与低分屏渲染实测后议）；丙·仿宋入 UI（列出即拒：自伤可读性）。
2. **聚珍新仿桌面内嵌授权**：C-1 发行门的第三张许可，独立于站面两张；未核清前壳侧文书轨以朱雀仿宋落地。
3. **标题轨具体字与字重阶**：B2-0 实测提值，架构核定。
