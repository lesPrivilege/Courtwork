# 架构裁决：sol A 阶段 [需架构拍板] 清单（2026-07-10）

## 1. 信源 tier/verified 的来源 → **core 事件流投影（已实现，B 阶段消费即可）**

UI 的判断正确：tier 不许由壳层猜。答案在 W6 刚交付的实现里：**信源等级台账在 core，`artifact_produced` 事件携带 evidenceGrades 投影**——引用级的 tier 与核验状态随事件到达 UI，artifact JSON 本体不含等级（等级是运行时工具调用的属性，不是数据的静态属性；将来若需持久化角标再走 schemas 提案，既有拍板）。北极星静态屏当时只能模拟，属预期；B 阶段对接真实事件流，此问题自然消解。**UI 只消费事件投影，零推断。**

## 2. 确认质量埋点 → **core 最小遥测事件 + 分析侧计算 + MVP 只告警不阻断**

采纳 sol 的替代方案：core 事件协议新增**最小审阅遥测事件**（打开条目 / 展开依据 / 提交处置，各带时间戳；不采原始鼠标键盘流），"秒批"识别等规则计算在分析侧，UI 只显示状态与消费门禁资格。**MVP 不做阻断**——律师确认快不等于错，结构性风险已由分层确认（防呆①）承接，因速度阻断专业人士是破坏信任的家长主义；埋点的用途是产品改进告警（docs/30 ③原意）。隐私归档：遥测事件属 docs/28 使用遥测档。落地为 **W6.1 微工单**（见 core SPEC TODO），不阻塞 B 阶段启动。

## 3. 语义色 AA → **双轨制：图形色与文本色分离**

采纳 sol 方案并定为通则：**法理之线线体保留高纯度原色**（线是图形元素，无正文 AA 要求，硬切的视觉冲击力保住）；**徽章/状态文字改用下压明度的 accessible fg**（琥珀 `#B45309` 起，红/绿/板岩同步复算至 12px AA）。tokens 每个语义色相拆双值：`*.graphic`（线/图形）与 `*.fg`（文本）。验收记录中错误的对比度数字更正。
