# 法理之线

法理之线是右栏审阅数据的处置状态载体：2px 竖线贴容器左缘，颜色 0ms 硬切。无线即没有需要表达的处置状态。

## 状态

| 条件 | token |
|---|---|
| 高危且待处理 | `line.danger` |
| 依据含未核验源 | `line.attention` |
| 已修订、待最终确认 | `line.revision` |
| 已确认 | `line.authority` |
| 已驳回 | `line.neutral` |

中低危待处理且依据已核验时无线；严重度由独立等级字段表达。并存优先级：`danger > attention > revision > authority > neutral`。

## 白名单

只允许用于：

1. 风险清单的处置行与门禁详情；
2. 修订预览的修订/新增内容；
3. 消费结构化 marker 的矛盾事件；
4. 其他带明确确认策略的右栏审阅卡。

中栏对话、通用 callout、导航、状态条、普通摘要、图谱节点和纯装饰元素一律无线。线色只能来自五个 `line.*` token，不得增加品牌色或第六种状态色。

## 与信源角标的关系

法理之线是卡片级处置状态；A/B/C 是引用级信源等级。未核验引用可令卡片进入 attention，但已核验引用不会自动令卡片变绿，只有用户确认才产生 authority。

## 行为

- 线本身不可点击；来源入口在引用上。
- 状态颜色不做渐变、呼吸或循环动画。
- 密集单行可省略域徽章，但若有处置状态仍保留 2px 线。
- 嵌套结构只有最外层语义容器有线。
- 打印可保留实色线；hover 效果不得承载信息。
