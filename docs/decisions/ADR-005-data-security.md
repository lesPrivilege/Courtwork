# ADR-005：状态、隐私与安全

- 状态：Accepted
- 日期：2026-07-13
- 来源：`8504aa9`、`335a351`、`a338361`、`00ec776`、`47d8697`

## 1. 状态与隔离

- 用户文件与应用状态分平面：原件/工作稿/产出在用户目录，artifact、账本、锚点、场景状态和缓存在 app data。
- 容器是隔离边界；数据访问必须携 container/case id，不提供无作用域全局读取。
- 权威态与缓存分离：事件与 artifact 版本不可变或可追溯，阅读视图/OCR 缓存可重建。

## 2. 隐私与凭证

- 案件内容永不训练；遥测不得含密钥、正文或可逆身份数据，且 opt-out 必须在发射点真实生效。
- 凭证只进系统钥匙串或明确网关，不扫第三方配置，不进入日志、事件流或前端明文。
- 企业私域权限采用上游 ACL 透传与检索时过滤，展示层遮挡不算权限控制。

## 3. 不可信输入的 MVP 最小集

1. OCR、宏和其他高风险解析留在 ingest 隔离进程；进程内纯库不得宣称自带沙箱。
2. 文档入口防 zip bomb、宏、XXE 与共享字符串膨胀，并统一复用安全预检。
3. web fetch 只允许 HTTP(S)，逐跳校验重定向、DNS 结果、私网/元数据地址、响应大小与 content type。
4. 进入生成节点的外部文本必须 spotlight；未消毒原文只可供 UI 展示。
5. TLS 或证书失败不得通过放宽校验静默重试。
6. 配置缺失、拦截与解析失败必须显式降级，不得返回伪成功空结果。
7. 宿主文件授权被拒绝、失效或目标卷不可用时必须显示具体原因与可执行下一步，不得伪装成空材料、空来源或成功读取。

## 4. 网络获取

### 4.1 间接内容同样不可信

搜索结果标题、摘要、重定向目标和短文本都可能承载间接提示注入；长度短不构成豁免。

### 4.2 SSRF 边界

协议、地址、DNS 解析和每次重定向都在连接前检查。私网、loopback、link-local、云元数据地址和等价 IPv6 表示一律拒绝。

### 4.3 Spotlighting

外部内容以随机边界与 data marking 包裹。消费方只能把 `spotlighted` 送入模型，并在系统层声明其为不可执行的待核验数据；`raw` 不得进入生成节点。

## 5. 后续演进

### 5.1 统一出口

当前“解析后校验、随后连接”仍有理论 DNS rebinding 时间窗。Stage 1 若真实风险或部署形态触发，应由服务端代理统一出口并钉住已校验的解析结果；MVP 不以自定义 dispatcher 假装形式化关闭该窗口。

## 后果

对外可以承诺本地优先、按容器隔离、不用于训练；不得宣称“绝对安全”。销毁容器属于用户不可逆决定，必须单独确认。
