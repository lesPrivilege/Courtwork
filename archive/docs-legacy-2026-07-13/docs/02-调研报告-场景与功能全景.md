# 法律垂类 AI 产品：场景与功能全景调研

- 调研日期：2026-07-09
- 调研方式：公开网络检索（产品官网、Legaltech Hub、LawSites/Artificial Lawyer、第三方测评、官方新闻稿），约 26 组中英文查询
- 用途：为面向中国律所/企业法务团队的垂类 agent 产品做场景注册表（scenario registry）规划参考
- 说明：定价与部分功能细节以第三方转述为主，标注来源；查不到的标"未公开/未查到"。第三方测评站（lawxyai、costbench、aivortex 等）数据可信度中等，仅作参考。

## 1. 场景全景图（统一场景分类树）

以下把海外 + 国内约 35 个产品的功能拆到叶子场景。标注格式：**[产品名]**=有该功能；(深)=该产品在此场景做得深/是招牌；(浅)=仅聊天式覆盖、无专门交互或工作流。

| 叶子场景 | 谁在做 / 深度 |
|---|---|

## 3. UI / 交互模式盘点

按"专门交互界面"归类，标注代表产品与成熟度（高/中/低）：

1. **表格审阅视图（document × question 矩阵）**——法律 AI 近两年最重要的 UI 发明。行=文档、列=问题/字段，单元格答案可点击溯源到原文。代表：**Legora Tabular Review**（招牌，高）、**Harvey Vault**（高）、**Hebbia Matrix**（高，且原生多模态）、Robin AI Reports（中）。国内：**无对标产品**。
2. **Word 插件内嵌（in-document）**——海外交易/合同类产品的默认形态：Spellbook、DraftWise、Legora、Robin AI、Luminance（红绿灯）、Definely、Lexis Create+、Clearbrief 全在 Word 里做标注/redline/引用面板。成熟度高。国内因 WPS 生态与网页习惯，走**自研网页编辑器**路线（幂律 PowerOffice：左文档右风险面板），Word/WPS 插件生态弱。
3. **左文档-右面板三栏审阅**——文档正文 + 右侧风险点/条款卡片/来源引用。代表：幂律 MeCheck（中-高）、法宝来签（中）、Definely（右栏条款检索）、Clearbrief（相邻窗格显示记录原文并高亮支持句，高）。
4. **红绿灯/多色风险标注**——Luminance Traffic Light（绿=合标、黄=偏离、红=重大偏离，红处给替代措辞一键插入，高）；MeCheck 高中低风险多色高亮（中-高）。
5. **超链接时间线（chronology 视图）**——Clearbrief（Word 内表格式时间线：日期/事件/记录引用超链接，高）、Wexler（事实图谱+时间线，事实为一等对象，高）、得理法搜（文书阅读时间轴，中）、Lexis Protégé（Vault 生成时间线，中）、Relativity aiR Case Strategy（新，中）。
6. **修订/比对视图（redline + 语义解读）**——Vincent 的 diff 每处变更附法律含义与相关判例（独特，中-高）；Spellbook/DraftWise 生成 Word 修订痕迹（高）；法宝来合同/MeCheck 比对（中）。
7. **引导式工作流/技能卡片（非聊天框）**——CoCounsel 的 skill 卡片、Vincent 的 20+ 预置 Workflow（引导式分步，明确反"开放式 prompt"）、Harvey Workflow Agents（可自建）、Lexis Protégé Agentic Skills。这是"场景按钮而非聊天框"的直接先例，海外成熟度已高。国内 AlphaGPT/法宝按功能模块分入口（中），但模块内仍以对话为主。
8. **审阅编码面板（eDiscovery）**——Everlaw/Relativity：文档列表+编码建议+AI 理由展示，防御性设计（高，但重）。
9. **仪表盘/台账**——Luminance 合规敞口仪表盘、Ironclad 合同分析看板、Yodex 法务管理台（中-高）。
10. **知识图谱/关系图**——检察系统"全要素知识图谱"（体制内）、智合AI 人物关系（低）；商用产品中**关系图谱作为交互产物的很少**，多停留在宣传层面。MetaLaw 思维导图输出（中-低）。
11. **结构化报告交付物**——国内特色：类案检索报告（法宝/得理/MetaLaw 一键生成）、法律意见书（法行宝）；海外：memo（Alexi）、demand package（EvenUp）。"以可下载报告为产物"是国内用户强偏好。
12. **客户/协作门户**——Legora Portal、Lexis Protégé Workrooms（协作工作间）、iCourt 诉讼协同。文件夹级多人协作 + AI 的组合尚无标杆产品。
13. **每步人工确认的流程护栏**——Garfield AI（每步需用户批准）、深圳法院系统（每环节勾选确认）。对低容错场景的交互范式有直接借鉴意义。

## 4. 空白与机会

对照目标定位——**面向普通法律从业者（非高认知律师）、卷宗文件夹级协作、OCR 前置摄取、官方数据接口、场景按钮而非聊天框**：

1. **国内没有"表格审阅"产品**。Legora Tabular Review / Harvey Vault / Hebbia Matrix 已验证该形态是批量文档场景的最佳交互，国内合同批量审查、尽调、批量案卷筛查均无对标。对卷宗文件夹场景，"行=卷内文件、列=要素问题"的矩阵视图可直接复用。
2. **卷宗级 OCR 摄取是真空地带**。国内律师拿到的是扫描件卷宗（倾斜、印章遮挡、手写批注），现状是律所自己拼 PaddleOCR/DeepSeek-OCR 私有化方案（2025 年多篇实战帖），没有商品化的"扔进一个文件夹→自动 OCR→结构化→可问答/可出时间线"产品。海外产品默认拿到的是 born-digital 文档，OCR 前置这一步在中国是刚需且无人做深。
3. **事实/时间线/矛盾检测（Wexler/Clearbrief 模式）国内几乎空白**。得理的时间轴只是单篇文书的阅读增强；智合AI 刑事阅卷笔记是雏形。跨卷宗多文档的事实抽取+溯源+供述矛盾检测，直接命中诉讼律师阅卷痛点，且交付物（带引用的时间线/矛盾清单）天然适合非高认知用户。
4. **"场景按钮"范式海外已验证、国内未做透**。CoCounsel skills 和 Vincent 引导式 workflow 明确证明：律师用户对预置场景入口的接受度高于开放聊天。国内产品虽有模块划分，但模块内仍是聊天框+自由 prompt；没有产品把"上传→选场景→结构化产出→人工确认"做成闭环流水线。Garfield AI 的每步批准机制是低认知用户产品的护栏范本。
5. **playbook 自动生成与立场化审查可结合**。DraftWise 能从历史文件自动归纳 playbook；通义法睿有甲/乙/中立立场审查。国内中小所没有历史 DMS 积累，机会在于"行业级预置 playbook + 按立场一键切换"，而非要求用户自建清单。
6. **官方数据接口是国内独有变量**。裁判文书网公开收紧后，权威数据向法信/法研（最高法系）、北大法宝等集中；海外 Harvey×LexisNexis、CoCounsel×Westlaw 证明"权威库联盟"是护城河的核心。面向律所的产品若能接通官方/准官方数据源（法信、法宝、地方司法数据），即是对 AlphaGPT/得理等自建库玩家的差异化。
7. **文件夹级多人协作缺标杆**。Legora Portal、Lexis Workrooms 刚起步；国内 iCourt 协同偏项目管理而非"围绕同一卷宗的 AI 协作"。"一个案件=一个共享文件夹=共享 AI 上下文+分工批注"没有人做成。
8. **监控预警类场景（法规更新、履约义务、续约）国内几乎无 AI 化产品**，Ironclad 2026 的三 agent 是方向参照；对企业法务客户是续费粘性最强的场景。
9. **风险提示**：合同审查单点（MeCheck/法宝来签/iTerms 已卷）、纯类案检索（法宝/得理/MetaLaw 已卷且数据壁垒高）不宜作为主打；Robin AI 团队被微软整体雇走、Spellbook 大幅涨价说明单点工具的商业脆弱性。
