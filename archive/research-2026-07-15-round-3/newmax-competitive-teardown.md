- **零独立信源**：知乎/V2EX/小红书/X/HN 多轮检索无任何第三方测评或讨论留痕；首页 12 条用户证言（含「陈律师」）仅见于官方营销页、库存头像、不可验证。
- **隐私叙事内部矛盾**：首页图注「ZERO TELEMETRY COLLECTION」，安全白皮书正文却自认收集匿名使用统计+崩溃报告+IP 城市级定位——营销层与实质层落差的现成案例（security 页，自报对照）。

- **法律只是画像不是语义**：企业版把「法律与咨询」列为五行业之一，卖点止于「私有化部署是合规底线」；对文书结构、引语锚点、事实等级、修订留痕零触碰。

## 架构定调（2026-07-17，产品负责人 + 架构）

1. **生成式 HTML 路线的瓶颈判断**（产品负责人定调）：HTML 交互不是先进范式，很早就在提；它抓住了结构化展示，但效益受 base model 能力瓶颈——普通提问被 agent 忙活半天画成不知何意味的 HTML（Manus、workbuddy 均有此通病）。病根不在渲染在裁决：模型在替用户决定「这值得一张界面」。**两线不冲突**：chat flow 引入 HTML 支持只是解耦的增加，当前不必做，后置为可选增项（届时仍须过架构拍板与残留/设计门）。
2. **同构对照第三例暂不记 NewMax**：其 GUI 机制查无实据，「生成式 HTML vs 编译式 schema 表」对照的实证案例仍以 Manus/workbuddy 为准；NewMax 记入的是另一条——**能力聚合 vs 秩序编译**（多模型路由+skill 市场+浏览器自动化的横向容器 vs 锚点/事实等级/确认账本的垂直裁决层），此对照成立。
3. **口径调整**：「本地优先」从卖点降格为准入门槛（两人团队几个月即可复刻全套姿态与话术，隐私叙事商品化在发生）；差异化收口到 NewMax 类产品零触碰的法律语义层。「零遥测图注 vs 白皮书自认收集」入口径素材库——静默降级零容忍的反面教材，我们的对应物是可验证机制（fail-closed 边界、readiness 单一口径）而非姿态文案。
4. **监测面**：错位竞争、非同一赛道；若其后续真做锚点式引证或文书结构理解再重估。个人版无审计日志这一点可直接用于对照演示（我们的确认账本 vs 其开关式权限）。
