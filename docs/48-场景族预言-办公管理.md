# 场景族预言：办公管理（2026-07-10）

状态：预言归档，全部零当期实现——机器已备（FileOpsPlan/四段式/sources 声明/门禁/scheduled 联动位/异步确认协议），只待场景声明。

## 场景族与 LLM 节点定位三问

财务报销、出差整理、日程管理等办公管理场景的设计工作收敛为三问：**输入是什么（schema）、LLM 在哪个节点说话（其余全为确定性步骤）、门禁开在哪**。示例：日程管理 LLM 节点仅一个（自然语言→结构化日程条目提案），其余纯 CRUD——docs/12 判据的极端例：步骤全可枚举，模型参与度越低可靠性越高。

## 语音与移动遥控

语音录日程归**通道网关移动侧**（企微/小程序，扫码绑定已拍板 docs/04）：手机采集→转文字→条目提案→异步确认（复用 W6 协议预留）。桌面端语音维持禁用态（docs/45）。小程序富交互后置，个人维护成本真实，Stage 1 后段。

## 飞书 = Stage 1 适配器优先候选（一鱼三吃）

IM 通道适配器 + 日程/审批数据源适配器（开放接口/CLI 同构能力）+ OA workflow 提交端（S7"自动到门禁为止"）。与企微并列白名单（企微胜律所渗透率、飞书胜新经济企业密度），先后由首批客户 IM 分布定。

## 纪律回指

对外提交类动作一律自动到确认门禁为止（docs/47）；来源授权显式、触发与声明解耦（docs/47 sources 节）；通道数据边界照 docs/04 红线（内容不出通道商可控范围）。

## 诉讼日程与期限族（2026-07-10 增补）

开庭提醒/举证期限/上诉期等**时效类条目单列高可靠等级**：多通道冗余推送（桌面+IM 网关）、确认回执（未读升级重试）、期限计算走确定性规则零 LLM 节点（误一次=执业事故，可靠性下限由规则保证）。庭审记录/庭前准备等按 docs/02 场景树货架位独立迭代。

## 性能重构范式（TS 验证 → Rust 重构）

确认为成熟范式，触发纪律：**性能数据驱动、不预做**——真实数据撞体验红线才立工单。重写保险 = 既有 golden files 与契约测试（换实现过同一套验收，语义漂移当场被抓）。候选优先级：reading-view 解析层、FileOpsPlan 哈希比对（纯函数、契约窄、Tauri 宿主本为 Rust）。品味驱动的重写是负资产。

## S10 候选·法律计算器族（2026-07-11 增补，据竞品全景）

竞品（类 chatbot 助手）铺 32 个计算器入口：民事赔偿 12（人损/借贷利息/工伤/违约金/继承/征拆等）+ 诉讼程序 4（诉讼费/刑期/审限/公告日期）+ 通用 16（财税/日期/大写金额等）。裁决四条：

1. **确定性工具族极端形态**：计算核心零 LLM 节点（与时效族同一可靠性裁决）；LLM 只做填表员——从卷宗/对话抽参数进 schema（SourceAnchor 溯源预填）→ 脚本算 → artifact 回填。docs/53 harness 可替换的又一实证。
2. **入口消失即交互优势**：竞品 32 入口工具箱 = chatbot 天花板（自找入口自填表）；我们按案由/上下文自动路由触发（trigger 三维现成），场景内铺已预填的表，用户只确认。
3. **隐藏机制、显式依据**：脚本机制零技术概念；计算结果必须带法条/标准出处 + 分项明细 + 参数溯源（律师要核数）——与竞品黑盒出数的第二差异点。
4. **壁垒在数据**：脚本设计时一遍写定（强模型代偿）；复利资产 = 法规参数库（LPR/各省赔偿标准年更/诉讼费率/伤残系数），按 docs/26 数据维护即场景长期维护。

排期：MVP 不做；调研单（32 项 × 参数 schema × 法规数据源 × 更新频率，一网打尽清单）成本低随时可派 subagent；刑期计算器注记为民刑兼顾差异化入口（竞品高亮位，Stage 2 刑事线参考）。

### S10 调研回册（2026-07-11，docs/56）

33 项全清单落档：17 张参数表按维护成本分级（零维护费率表 → 省级年更 → 城市级按需）；第一批 7 项建议（诉讼费/迟延履行/借贷利息/法律期间引擎【与时效族共内核】/劳动三件套/工伤最小版/人民币大写——共同特征：数据依赖 ≤3 张表）；人损旗舰推第二批等省级三指标表。三纪律采纳：CalcArtifact schema 先行（分项+依据+SourceAnchor）；法条逐项核验（民诉法 2023 修正条号漂移已抓一例：涉外公告送达 60 日）；**裁量项永不出单值**（出区间——模型只生成不裁决在计算域的表达）。
